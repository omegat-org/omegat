Principles

Welcome to OmegaT, a computer-assisted translation (CAT) tool that offers great flexibility, performance and robustness, as well as features beyond those of commercial offerings, even for beginning translators.

Translation project

An OmegaT translation project consists of a number of resources stored in separate folders, of a omegat.project file that stores the location of all the resources and of a project_save.tmx translation memory file that stores all the translations made while working on that project.

Translators can easily modify the contents or location of those resources at any time during the translation.

By default, resources include the project’s translation memory, which is automatically created upon project creation, and the files to translate. They can also include reference translation memories, glossaries, and other reference files.

See Project Folder for details.

Translation memories

Translators do not have to create a translation memory or associate one to a new project. OmegaT takes care of that automatically .

Translators can very easily reuse an existing projectʼs translation memory by simply adding new translation files to the old project, or by copying the old translation memory to a new project.

There is no limit to the number of translation memories that can be used as reference in a translation project, and OmegaT makes it very easy to assign different levels of priority to these additional memories.

See Use translation memories for details.

Collaboration

Simple settings enable translators to easily share resources and work together on a project.

The team functionality offers industrial grade robustness and flexibility to synchronize project resources among team members, while allowing easy resolution of conflicts between different translations.

See Share TMs for details.

Made for translators

OmegaT is designed not only to facilitate the translation process, but also to ensure the translator does not have to worry about losing data.

Note

You have not used OmegaT yet, but you remember the times you were translating in your word processor and a computer glitch wiped out the book translation you had been working on for the last six months, or the report that was due in one hour.

You're considering OmegaT, but…

How can OmegaT help against such glitches?

Not to worry. The robust design of OmegaT makes it extremely unlikely that you have lost more than a few minutes of work.

Even if things look very broken, do not panic. Just follow the simple steps described in Troubleshoot issues.

And before you ever need to follow those steps, OmegaT will allow you to focus on actually translating, and hopefully help you become a better translator.

Shortcuts

OmegaT looks a bit stern, if not outright unfriendly, to people who are used to button based interfaces. Buttons are very few in OmegaT. Most of the action takes place with the help of shortcuts associated to menu items.

Shortcuts are ubiquitous in this guide and you should be able very shortly to remember the few that are useful in the course of a standard translation day.

To avoid duplication of shortcut description, we have adopted the following notation in this guide:

Example 1. Single letters are used as a common identifier for modifiers.

In this guide: C + E

On Windows and Linux: Control + E

On macOS: Command + E


See Menus for details.

Create a new project

Warning

If you have not yet installed OmegaT, check Install OmegaT and Run OmegaT.

To start using OmegaT, you first have to create a project to hold all your files.

Use New... C + S + N , select an empty folder, or create a new one with the name of your project.

An OmegaT project is simply a set of folders and folders that holds all your default resources.

The Project Properties dialog opens after you have named your project.

At the top of that dialog, select the languages of your source and translated files, set the various options, and click the OK button to continue.

After you have created the project, you can use Properties... C + E to come back to this dialog as necessary.

Note

You can modify the structure of a project at any time after you have started translating. In the dialog, just tell OmegaT what new resource folders to use. It will then just ask you to reload the project and start using the new locations.

The Source Files dialog opens next.

Click on Add Files... or Add Online MediaWiki Page... and add source files.

If you don’t remember where your project is, you can use Access Project Contents from the Project menu.

If a file does not appear into the Editor pane, then it could be that it is empty, or in a format that is not supported by OmegaT. Use Global File Filters... to open the list of supported file formats. If the format is not there, use third party plugins or convert the file to a format that is supported. See Support other formats for details.

Manage your segments

The default setting for a translation project is to create sentences by splitting the paragraph blocks found in the document. See Sentence-level Segmenting for details.

Segments that are within paragraph can be further segmented, or "unsegmented" (merged).

To further segment them, add a segmentation rule, to merge them, add an exception rule. See Segmentation for details.

Note

Since a standard space character is often used as a boundary for a segment, if you want to merge two parts that are separated by a space character,

  1. go to the source file with Access Project Contents,

  2. find the location of that space,

  3. replace it by anything that is not a standard space but looks like a standard space, like a "non breakable space" for example,

  4. save your modifications,

  5. go back to OmegaT and Reload F5 .

Translate your files

The Editor pane is the place where you type your translation.

Note

To see how efficient you can be with shortcuts, see A shortcut based workflow.

After you have typed the translation of a segment, use Next Untranslated Segment C + U to move to the next untranslated segment.

Once you leave the segment, its translation is automatically stored into project_save.tmx and will be used as reference for the rest of your translation.

See the Go To menu for more navigation options.

Note

OmegaT does not have different statuses for translated segments. Either a segment is translated, or it is not. If you need a separate review process, you can for example use the Notes panes for problematic segments, or use Search... C + F to filter segments based on criteria that you set, and review only those, etc.

Use Create Glossary Entry... C + S + G to add terms to your glossary.

Use Mark Glossary Matches in the View menu and Auto-Completion to manage glossary matches.

When you enter a segment where terms match terms in your glossaries, the terms and their translations will be displayed in the Glossary pane.

When you enter a segment that matches some of the content of the project translation memories, OmegaT displays the matches in the Fuzzy Matches pane.

Use the Edit menu to manage translation memory matches and perform other various useful functions.

With Search... C + F , you can search the whole project for terms, sentence fragments and so on, using exact, keyword, or regular expression searches, as well as narrow the scope of the search based on information such as author or date.

You can also focus only on the results by filtering them in the Editor. See Text Search for details.

You can replace terms with Replace... C + K using a similar dialog. See Text Replace for details.

You can leave notes to yourself or team members in the Notes pane while you translate, and go back to annotated segments with Next Note or Previous Note in the Go To menu, or from the Text Search window.

The Context menu gives easy access to some frequently used functions. The Auto-completion menu is also available in the Editor pane.

Manage your tags

If the documents you translate use bold, italics, or other decorative elements, OmegaT will convert them into tags that enclose the decorated text. Such files include LibreOffice files, HTML files, Microsoft Office files, etc. Documents will often also contain tags that have nothing to do with decorations, but are nevertheless important in the source files (and, consequently, in the translated files as well).

A source sentence might look like:

Example 2. Sentence with decorative formatting

OmegaT is an easy-to-use program for eager translators


Depending on the original format, OmegaT, however, will present this sentence in a fashion similar to this one:

Example 3. How decorations are displayed in OmegaT

OmegaT is an <t0/>easy to use<t1/> program for <t2/>eager<t3/> translators


The tags in OmegaT are grayed, making them easy to recognize. They are protected to prevent you from modifying their contents, but you can delete them, enter them by hand or move them around in the target sentence. See Insert Missing Source Tags C + S + T and Insert Next Missing Tag C + T for details.

You can define custom tags that will be managed in the same manner. See Custom tags for details. You can also define strings that OmegaT should warn you about if you leave them in a translation. See Flagged text for details.

If you make mistakes when you manage the tags, your translated files might fail to open. Use Check Issues... C + S + V to check your tags before you generate your translated files.

Warning

If a file refuses to open in LibreOffice, Word or other application, it is often due to errors in the management of OmegaT tags. Return to OmegaT, check your document for tag issues and recreate the translated file.

Review your translation

Feel free to modify the Panes layout and View options when you review the translation.

Use Check Issues... C + S + V to make various checks on the contents.

Create the translated files

Whenever you want to see what your translation will look like in its final format, use Create Translated Files C + D .

Warning

Any time you create the translated files, any previously created files with the same name will be overwritten. If you want to keep files from being overwritten, change their name after you created them, or automate the name modification with Edit.

Your translations are always stored in project_save.tmx, the project translation memory, located in the omegat folder of your project.

Deleting the contents of the target folder will never impact your translations.

The documents will be created in the target folder of your project, and unless you have modified the names of the files that were previously created, they will be overwritten by the process...

Once the translated files have been created, you can open them in their original application to check their contents and eventually further modify them. Such external modifications will not be reflected in OmegaT.

Modifications made outside OmegaT can be reinserted in OmegaT (and thus in the project translation memory, to keep it up to date) either by entering them manually in the relevant segments, or by aligning the original file and the final translated file to create a new translation memory (TMX) file that will be put in the tm/enforce folder of your project folder. See Align Files for details.

Manage your projects

  • Translation projects are just folders containing files. You can create a new project for each new job, or you can add new source files to an existing project at any time.

  • You can access and modify the project settings with Properties... C + E . To see the list of files in the project, use Source Files... C + L . You can access all the resource contents at any time with Access Project Contents.

  • Every time you create the translated files (see Create the translated files, above), OmegaT creates three translation memories that contain the state of the translation of the files currently in the source folder. See Exported Translation Memories Folder for details.

For a more comprehensive introduction see OmegaT for beginners on the OmegaT web site. If you need assistance with any aspect of OmegaT, feel free to join the OmegaT user groups.
