Menus are where you access most of OmegaT's functions. Most of the menu items are associated with a keyboard shortcut.

Menu items that do not have a shortcut indicated next to them have no default shortcut assigned by default. See Customization if you want to modify or add shortcuts.

Note

Key shortcut description reminder:

Linux/Windows Key identifier macOS
Shift S shift or ⇧
Ctrl or Control C command or ⌘
Alt A alt / option or ⌥
  Ctrl control or ⌃

Project

Example 11. New...

In this guide: C + S + N

On Windows and Linux: Control + Shift + N

On macOS: Shift + Command + N


New... C + S + N

Creates and opens a new project. See Create a new project for details.

Download Team Project...

Creates a local copy of a remote OmegaT project. See Use a Team Project for details.

Open... C + O

Opens a previously created project.

Open Recent...

Gives access to the last ten edited projects. Clicking one will save the current project, close it and open the selected project.

Use Clear Menu function to delete the list of recent projects.

Reload F5

Reloads the project to take external changes in source files and project settings into account.

Note

New translation memories placed in the tm folder while translating are loaded automatically as soon as the cursor moves from one segment to another. Similarly, the contents of the glossary folder are automatically recognized and do not require reloading the project.

Close C + S + W

Saves the project translation memory and closes the project. See project_save.tmx for details.

Save C + S

Saves the project translation memory. See project_save.tmx for details.

OmegaT automatically saves translations every 3 minutes as well as when you close the project or quit OmegaT. See Project data saving interval for details.

Add Files...

Copies the selected files to the source folder and reloads the project to take the new files into account.

Add Online MediaWiki Page...

Opens a dialog where you can paste the URL of the Mediawiki page you want to translate. The source data of the page will be copied into the source folder, as a text file, with the .utf8 extension.

Commit Source Files

Warning

This function is specific to team projects. Only the team project manager should use this function . See Use a Team Project for details.

Uploads locally added or modified files in the source folder to the team project repository.

Commit Target Files

Warning

This function is specific to team projects. Use this function only if the team project manager requested so . See Use a Team Project for details.

Uploads locally created in the target folder to the team project repository.

Create Translated Files C + D

Creates the target files based on your translation. The created target files are located in the target folder.

Note

You can block target file creation if tag issues are found. See Block the creation of translated files with tag issues for details.

Create Current Translated File C + S + D

Creates the target file corresponding to source file currently being translated. The resulting file is located in the target folder.

Open MED Project...

Opens a project package in the MED format defined by the EU Directorate-General for Translation.

Create MED Projects...

Converts the current project into a MED package.

Properties... C + E

Displays the Project Properties dialog to edit the project languages and folder locations.

Source Files... C + L

Opens the Source Files window.

Access Project Contents

Gives access to the various project folders. See Project Folder for more details.

In addition, there are three menu entries that directly open the current source or target files, or the writable glossary. The files are opened in the default application set by the operating system. The options are grayed out if the files do not exist.

Restart

Saves the project and restarts OmegaT. You are asked to confirm that you really want to quit if you have not yet saved the project.

Edit

Example 12. Undo Last Action

In this guide: C + Z

On Windows and Linux: Control + Z

On macOS: Command + Z


Undo Last Action C + Z

Restores the status before the last edit made in the current segment. This command cannot restore that status after the user moves to a different segment.

Redo Last Action C + Y

Restores the status before the last editing action was canceled. This command cannot restore that status after the user moves to a different segment.

Replace With Match or Selection C + R

Replaces the entire target segment with the currently selected fuzzy match (by default the first match is selected). See Fuzzy Matches for details.

Any text selected in the Fuzzy Matches pane will have priority over the selected match and will replace the entire target segment.

Insert Match or Selection C + I

Inserts the currently selected fuzzy match at the cursor position. If part of the target segment has been selected, this function overwrites the selected portion. See Fuzzy Matches for details.

Any text selected in the Fuzzy Matches pane will have priority over the selected match and be inserted at the cursor position.

Replace with Source C + S + R

Replaces the entire target segment with the segment source.

Insert Source C + S + I

Inserts the segment source at the cursor position.

Replace with Machine Translation C + M

Replaces the target segment with the translation, provided by the selected Machine Translation service.

If Automatically fetch translations is disabled, use this action a first time to fetch the translation and a second time to insert it.

No action is taken if no Machine Translation service has been activated (in Machine Translation). See Machine Translation for details.

Insert Missing Source Tags C + S + T

Inserts the remaining missing source tags at the cursor position.

Insert Next Missing Tag C + T

Inserts the next missing tag at the cursor position.

Export Selection C + C + S

Exports the current selection to a text file for processing. If no text has been selected, the current source segment is written to this file. To remain consistent with usual clipboard behavior this file is not emptied when the user exits OmegaT. The exported contents are copied to the file selection.txt located in the Configuration Folder folder.

Create Glossary Entry... C + S + G

Allows the user to create an entry in glossary.txt.

There are two ways to use this function. There first is to call the dialog and then manually fill in the various fields. The second is to select items in one of the OmegaT panes and sequentially call the dialog after each selection to have OmegaT enter it automatically in the various fields:

  1. Select a text string in any pane

    Press C + S + G

    The selection will be entered in the Source term field.

  2. Select a text string in any pane

    Press C + S + G

    The selection will be entered in the Target term field.

  3. [Optional] Select a text string in any pane

    Press C + S + G

    The selection will be entered in the Comment field. You can also enter your comment manually.

  4. Hit Enter or click on OK

Search... C + F

Opens a new Text Search.

If you select a text string (in any pane) before calling the function, the text will be pasted by default in the Search for: field.

C + S + F reuses the most recent still open Search window (instead of opening a new one).

Replace... C + K

Opens a new Text Replace.

If you select a text string (in any pane) before calling the function, the text will be pasted by default in the Search for: field.

Search Dictionaries

If Search automatically is disabled, this function allows you to search for the selected word in the dictionaries.

Switch case to

Changes the case of the selected text in the target segment to the selected option, or cycles through the options. If no text is selected, OmegaT selects the word starting with the character immediately to the right of the cursor.

  • lower case

    all letters in lower case

  • UPPER CASE

    ALL LETTERS IN UPPER CASE

  • Title Case

    Almost all Letters in Lower Case

  • Sentence case

    Only the first letter in upper case.

  • Cycle S + F3

Select Match

Selects the previous, next or nth fuzzy match displayed in the fuzzy match pane to replace or insert it to the segment.

  • Select Previous Match A + ↑

  • Select Next Match A + ↓

  • Select Match #1 C + 1

  • Select Match #2 C + 2

  • Select Match #3 C + 3

  • Select Match #4 C + 4

  • Select Match #5 C + 5

Insert Unicode Control Character

Inserts the selected Unicode control character. See Manage RTL Languages for details.

  • Left-to-Right Mark (LRM U+200E)

  • Right-to-Left Mark (RLM U+200F)

  • Left-to-Right Embedding (LRE U+202A)

  • Right-to-Left Embedding (RLE U+202B)

  • Pop Directional Formatting (PDF U+202C)

Use as Default Translation

If several alternative translations are available for the active segment, you can label the alternative selected as the default translation. The entry will be greyed if only one translation is available.

Create Alternative Translation

Segments that are one and the same segment may, depending on the context, require different translations. If the current translation does not apply, select this menu item and enter the alternative translation.

Remove Translation

Deletes the current translation and sets the segment as untranslated.

Set Empty Translation

Define the translation as empty. In the target document, nothing will appear for this segment. In the Editor, the translation is displayed as <EMPTY>.

Register Identical Translation C + S + S

Use this command to register a translation as identical to the source, even if Allow translation to be equal to source is not checked.

Example 13. Next Untranslated Segment

In this guide: C + U

On Windows and Linux: Control + U

On macOS: Command + U


Next Untranslated Segment C + U

Moves to the next segment with no target language entry in the translation memory.

Next Translated Segment C + S + U

Moves to the next already translated segment, ignoring untranslated segments.

Next Segment C + N

Also: Enter

Moves to the next segment, or to the first segment of the next file if the current segment is the last one in the file.

Note

TAB can be used as an alternate shortcut if Use TAB to Advance is selected.

This is useful for character entry systems that use Enter to validate input.

Previous Segment C + P

Also: C + Enter

Moves to the previous segment, or to the last segment of the previous file if the current segment is the first one in the file.

Note

C + TAB can be used as an alternate shortcut if Use TAB to Advance is selected.

This is useful for character entry systems that use Enter to validate input.

Segment number... C + J

Jumps to the segment whose call-out or number is entered.

Next Note

Moves to the next segment with a note.

Previous Note

Moves to the previous segment with a note.

Next Unique Segment C + S + Q

Moves to the next unique segment.

Source of Selected Match C + S + M

Moves to the segment that corresponds to the currently selected match in the Fuzzy matches pane.

Forward in History C + S + N

OmegaT remembers your segment navigation history.

This command allows you to move forward one segment at a time to segments that you have previously visited with Back in History C + S + P .

Back in History C + S + P

OmegaT remembers your segment navigation history.

This command allows you to move backward one segment at a time to segments that you have previously visited.

View

Mark Translated Segments

If checked, translated segments will be marked in yellow.

Mark Untranslated Segments

If checked, untranslated segments will be marked in violet.

Mark Paragraph Delimitations

If checked, separations between paragraphs in the source document are indicated visually. See Paragraph delimitation format.

Display Source Segments

If checked, source segments will be shown and marked in green. If not checked, source segments will not be shown.

Mark Non-Unique Segments

If checked, non-unique segments will be marked in pale gray.

Mark Segments with Notes

If checked, segments with notes will be marked in cyan. This marking has priority over Mark Translated Segments and Mark Untranslated Segments.

Mark Non-Breakable Spaces

If checked, non-breakable spaces will be displayed with a gray background.

Mark Whitespace

If checked, white spaces will be displayed with a small dot.

Mark Bidirectional Algorithm Control Characters

This option displays bidirectional control characters

Mark Auto-Populated Segments

If checked, segments that have been auto-populated (from TMXs placed in tm/auto for example) are marked in various colors.

That color marking is shown as long as Save auto-populated status is checked.

Normal translations inserted from the tm/auto folder are marked in orange. Other translations specifically identified in the TMX, can be marked in different colors. See the Request For Enhancement for technical details.

Mark Glossary Matches

Underlines source terms that have a match in the glossaries. Hovering on an underlined term displays the matching target term. Right-clicking on an underlined term displays the matching target term, which can be selected and inserted at the cursor position in the target text.

Language Checker

Underlines parts of the target where LanguageTool has found errors. See LanguageTool for details.

Aggressive Font Fallback

Check this option in case OmegaT does not display certain glyphs properly (even if the fonts containing the relevant glyphs are installed on your machine).

Note

On Windows this manual font fallback mechanism has been known to interfere with the standard font fallback, causing undesirable font substitutions. For that reason, it is OFF by default.

Modification Info

Setting the Display Modification option to Current segment will display the time and the author of the last change made to the current segment. Setting it to All segments shows this information for all segments, while None turns this option off.

You can adapt the displayed information to your needs. See Customize segment modification information for details.

Restore Main Window

Restores the OmegaT main window layout to its default values, with the Editor pane on the left, the Fuzzy Matches pane at the top right and the Glossary pane at the bottom right. All other panes are minimized.

Note

Use this function when you run into problems while rearranging the window layout.

Example 14. Check Issues...

In this manual: C + S + V

On Windows and Linux: Control + Shift + V

On macOS: Shift + Command + V


Check Issues... C + S + V

This quality assurance tool carries out all automatic checks in one go and displays the results in a window.

Four types of issues can be detected:

  • Tag Issues (always selected): detects missing or displaced tags, including custom tags and flagged text. See Tag Processing for details. The option is always selected.

  • Spelling Issues (optional): detects spelling mistakes. Only works if a spelling dictionary is installed. See Spellchecker for details.

  • Terminology Issues (optional): detects all the glossary items that are not properly translated. See Glossary for details.

  • LanguageTool Issues (optional): detects grammatical or typographical issues. See LanguageTool for details.

The results are presented as a table in which:

  • double-clicking a row activates the corresponding segment in the Editor pane,

  • clicking a column header changes the sort order for that column,

  • selecting a row, or moving the mouse over it, displays a context menu icon in the last column; clicking that icon presents actions available to correct or ignore the error.

    Note

    You can force issue checking each time you leave a segment (see Check issues when leaving a segment) and block translated files creation if tag issues are found (see Block the creation of translated files with tag issues).

Check Issues for Current File

As above, but only for the document currently displayed in the Editor pane.

Statistics

Opens a new window and displays project statistics such as the overall project word count or segment totals, and totals for every file in the project.

The data is also saved as a text file in omegat.

Note

Word count, segment total and other items can be influenced by the following factors:

  • File filter settings: some filters allow for extra parts to be considered for translation. See Options for details.

  • Segmentation rules: different rules will generate a different number of segment. See Segmentation for details.

  • Tags: OmegaT tags are never counted in the statistics but in some cases can split terms and create discrepencies in the way OmegaT counts them. See Remove Tags for details.

  • Custom tags and flagged text: as for OmegaT tags, they are not counted by default in the statistics, but you can have OmegaT count them as words. See Count flagged text and custom tags in statistics for details.

Match Statistics

Displays the Match Statistics for the project: the number of repetitions, exact matches, fuzzy matches and no-matches for segments, words and characters.

The data is also saved as a text file in omegat.

Match Statistics per File

Displays the Match Statistics for each file of the project: the number of repetitions, exact matches, fuzzy matches and no-matches for segments, words and characters.

The data is also saved as a text file in omegat.

Align Files...

Select the two files to align (the source file and its translation) and click OK to open the Align Files window.

The supported file formats depend on your project setting. See File Filters for details.

The source and target files can have different formats (for example, a .docx file can be aligned with a .pdf file).

Scripting...

Opens the Scripting window, which allows you to set the location where scripts are stored, as well was write and run scripts, and assign them a shortcut.

The list below this item displays 12 potential slots for scripts. Clicking on an assigned slot launches the script assigned to that slot.

Note

To assign a number to a script:

  1. Open the Scripting window

  2. Select the script you want from the list on the left.

  3. At the bottom of the window, right-click on an unassigned number and select Add Script.

External search commands

If you have defined external searches in Global External Searches, their names are listed and accessible here.
