Principles

Manipulation

The main window consists of several panes, described here, the main Menus, and a Status bar. You can change the position of a pane, or even undock it as a separate window, by clicking and holding its name and dragging it.

The top right corner of a pane will show different icons depending on the type of pane and its current status:

Table 1. Pane widgets
Lists the various extra actions available in the pane.
Minimizes the pane, so that it only shows as a tab at the bottom of the window.
Maximizes the pane
Restores a maximized pane to its previous layout.
Undocks the pane from the main window.
Returns the pane to the main window.

You can group panes in the same area of the window. Doing so shows a tab with the name of the panes the top of the group. Panes can be resized by dragging the separators between them.

Note

If you lost track of the changes you made to the user interface, and cannot see some panes, use Restore Main Window in the View menu any time to go back to the default layout.

Context menus

Some panes have a context menu. You can call that menu using the standard method for your OS, or with the OmegaT defined shortcuts:

  • On Windows and Linux: Menu

  • On macOS: S + Esc

Dragging and dropping

Files can be dragged and dropped on certain panes, with the following results:

Editor pane

Dropping an OmegaT project file (omegat.project) or a full project here will open the corresponding project after closing the currently open project, if any.

Dropping other files will copy them to the source folder. This also applies to the Source Files window.

Fuzzy Matches pane

Dropping .tmx files here will copy them to the tm folder.

Glossary pane

Dropping files with a recognized glossary extension (.txt, .tab, etc.) here will copy them to the glossary folder.

Editor

This is where you type and edit your translations.

The text is split into numbered segments. You can scroll through the document and double-click on any segment to open and edit it.

Example 5. Default editor writing space

— ¶ —————————————————————

Remove Tags
|Supprimer les balises<segment 2148 ¶>

— ¶ —————————————————————

When enabled, all the formatting tags are removed from source segments.

  • The first line, in bold characters against a green background, is the source text to translate.

  • The translation field is the second line, between the left border and the segment marker on the right. The translation field is where you enter your translation.

  • The segment marker here is <segment 2148 ¶>.

    2148 is the number of the segment in the project and the ¶ symbol indicates that the segment is the first segment in a paragraph.

  • The cursor is by default locked into the translation field.

    Use F2 to unlock the cursor and move it beyond the translation field, and to make it come back into the translation field when your are done.

  • The line below the translation field is a different segment.

  • Use Next Segment C + N or other navigation commands to:

    • register the translation field contents in the project translation memory, and

    • enter the next segment to translate.

    See the Go To menu for details.

Note

Modify the Editor display to suit your workflow. See the Editor preferences for details.

Modifications can include:

  • Font

  • Mark Paragraph Delimitations

  • Paragraph delimitation format

  • Modification Info

  • Display Source Segments

  • Mark Translated Segments

  • Leave the segment empty

  • Insert the best fuzzy match

  • Flagged text

Empty translations

If you leave the translation field empty, OmegaT will treat the segment as untranslated and keep it in the original language when it creates the translated documents.

Translations equal to the source

OmegaT can store translations that are identical to the source text. This can be useful for documents that contain trademarks, names or other proper nouns, or sections in a third language that do not require translation.

Dragging text

It is possible to drag text from anywhere in the main window and drop it into the translation field. Text dragged from outside the translation field is copied, while text dragged from within the translation field is moved.

Overwriting

By default, typing text inserts the new text after the cursor in the typing direction. To overwrite the existing text, use the following keys:

  • On Windows and Linux: INSERT

  • On macOS: F3

The writing mode (overwrite - OVR or insert - INS) is shown at the right of the Status bar. Use the same key to switch back and forth between each mode.

Cursor lock

By default, the cursor is locked into the translation field, and the arrow keys cannot be used to move in the source text.

Pressing F2 unlocks the cursor and makes it possible to use the arrow keys to move in the source text (or anywhere else in the editor). This allows you to select text using the keyboard.

The cursor status (locked - LCK) or unlocked - UNL) is shown at the right of the Status bar. Use the same key to lock the cursor again.

Context menu

Default shortcut:

  • On Windows and Linux: Menu

  • On macOS: S + Esc

You can also bring up the menu by right-clicking the mouse ( C  +  click on macOS).

The Editor pane context menu offers:

  • Cut, Copy, Paste functions,

  • Go To Segment (to jump to the segment under the pointer),

  • Search Dictionaries (to search for the selected term in an installed dictionary),

  • Add Glossary Entry (equivalent to Create Glossary Entry... C + S + G ),

and other functions available from the Edit menu.

If you have selected text and defined entries in Global External Searches, they will also be displayed in the menu.

Auto-completion menu

Default shortcut:

  • On Windows and Linux: C + Space

  • On macOS: Esc

This menu offers the auto-completion options set in Auto-Completion.

Fuzzy Matches

The match viewer shows segments from your translation memories that are the most similar to the segment you are currently translating.

Matches are found both in the internal project translation memory that you create in real time as you translate the project, and in external memories you have imported from previous projects or received from third parties. See Use translation memories for details.

When you enter a segment, the first fuzzy match (the one with the highest match percentage) is automatically selected. You can press C + 2, 3, 4, 5 to select a different match.

Example 6. The first match

1. Configuration Folder
Dossier de configuration
<50/50/66% >


The selected fuzzy match is highlighted in bold, with words missing from the segment you are translating shown in blue, and adjacent words shown in green.

The match also displays three percentages.

The three matching percentages have the following meaning, in order:

  • Percentage calculated with stemming based on the source language tokenizer, and ignoring tags and numbers (generally the highest).

  • Percentage calculated without stemming, and still ignoring tags and numbers (generally slightly lower).

  • Percentage calculated on the entire text, including tags and numbers (generally the lowest).

You can change how matches are displayed or sorted. See the TM Matches preferences for details.

Use Replace With Match or Selection C + R to replace the translation field with the match, or Insert Match or Selection C + I to insert it at the cursor position.

Glossary

The Glossary pane shows terms from your glossary files that correspond to terms in the current segment. The glossary files are located in the project’s glossary folder.

The source term is displayed before the “=” sign and the target terms after it. If the entry contains comments, they are numbered and displayed on separate lines below the terms. Terms found in the writable glossary are shown in bold, while those from other glossaries are presented as regular text.

Example 7. Glossary matches

Project = Projet
1. Projet de traduction

Folder = Dossier


The glossary file name is shown as a tooltip when hovering over the terms.

Hyperlinks can be present in the comment field of an entry and can be clicked upon for further reference.

Links to local files (file:///PATH/filename) open in the associated program. Web addresses (http://...) open in the default browser.

You can select a term in the glossary pane and right click on it to insert it into the translation.

If you have activated Mark Glossary Matches in the View menu, you can right-click on the highlighted word in the source segment to open a pop-up menu listing the translations available in your glossaries. Selecting one will insert it in the segment at the current cursor position.

You can use the glossary preferences to fine-tune how the glossaries are displayed. See the Glossary preferences for details.

Modifications to any glossary file in the glossary folder are immediately recognized by OmegaT and displayed in the glossary pane.

OmegaT includes one writable glossary file per project. Use Create Glossary Entry... C + S + G in the Edit menu to add terms to that file.

See the Glossaries appendix for details.

Dictionary

The Dictionary pane shows terms from your dictionary files that correspond to terms in the current segment. The dictionary files are located in the project’s dictionary folder. See the Dictionary preferences for details.

Machine Translation

The machine translation pane, when opened, contains the suggestions produced for the current segment by one or more enabled translation engines. When suggestions from more than one engine are available,

  • the name of the engine will appear after the translated text,

  • the results are sorted in alphabetical order of engine names,

  • and the currently selected translation is highlighted.

Use Replace with Machine Translation C + M to replace the translation of the current segment with the selected translation engine.

The "Origin" property of the current segment is assigned the selected engine name.

If there is a connection or authentication error with a machine translation engine, the status bar at the bottom of the window will briefly display status message.

Make sure the service is enabled and your credentials are entered correctly. If they are, you may need to contact the service provider for assistance. Similarly, an error such as Unable to parse translation service response suggests that there may be a communication problem between your system and the provider's server.

See the Machine Translation preferences for details.

Multiple Translations

A source segment that appears in multiple locations in the project may require several different translations depending on context.

If the current translation of the segment is not suitable, you can use the Create Alternative Translation command in the Edit menu to specify an alternative translation. The entered translation will then be treated as an alternative translation of the source segment.

You can define one of the alternatives—the most common or likely, for instance—as the default translation with the Use as Default Translation command.

Notes

The Notes pane is an editable space where you can add notes to the currently active segment.

This lets you come back to it later to review the translation, check that alternative translations are correct or, in shared projects, ask colleagues for their opinion, for example.

You can browse through the notes using Next Note and Previous Note.

Note

The notes are stored in the translation memory of the project. They are not stored in the translated files.

Comments

Some file formats, such as the PO format, can provide the translator with comments to add context to the translation. Such comments are displayed here.

Note

The Comments pane is not editable. It only displays information that is included in the file.

To write your own comments about a translation, use the Notes pane instead.

Segment properties

All segments have properties. The properties are displayed in this pane.

Common properties include:

Is duplicate

indicates that a segment is a duplicate and specifies whether the segment is the FIRST or not.

File

indicates the file where the segment is stored.

Note

The Properties pane is not editable. It only displays information that is related to the segment.

Status bar

The bottom of the main window is the status bar.

The left side displays information about actions taken in the project or error messages.

Example 8. Status bar messages examples

→ OmegaT is opening a team project and synchronizes with the remote repository.

→ OmegaT opens the last edited segment from the last session.

→ OmegaT has recently saved the project.

Translated files created

→ Files were created by the user.


The right side displays the Cursor lock and the Overwriting status followed by progress information.

Click on the figures to alternate between displaying progress as numbers or percentages.

Example 9. Status bar progress information

LCK | OVR 257/271 (12292/154896, 177692) 35/0

In the file

257 = number of translated segments / 271 = total number of segments

In the project

12292 = number of unique translated segments / 154896 = total number of unique segments, (177692) = total number of segments

In the segment

35 = number of characters in the source / 0 = number of characters in the target area so far


Example 10. Status bar progress information, percentage mode

LCK | OVR 94.8% (14 left) / 7.9% (142,604 left), 177,692 35/0

In the file

94.8% = % of translated segments / 14 = number of untranslated segments

In the project

7.9% = % of translated segments / 142,604 = number of untranslated segments, (177,692) = total number of segments

In the segment

35 = number of characters in the source / 0 = number of characters in the target area so far
