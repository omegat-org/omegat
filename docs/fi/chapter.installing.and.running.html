Windows-käyttäjät

Oikeanlaisen paketin lataaminen

Onko koneellasi Oraclen Java 8 JRE -yhteensopiva Java-ympäristö?

  • Kyllä: Lataa tiedosto OmegaT_4.n.n_Windows_without_JRE.exe.

  • Ei / En tiedä: Lataa tiedosto OmegaT_4.n.n_Windows.exe.

    Tässä paketissa on mukana Oraclen täydellinen Java-ajoympäristö. Tämä JRE ei vaikuta muihin Java-ympäristöihin, joita koneellesi on mahdollisesti jo asennettu.

OmegaT:n asentaminen

Asenna OmegaT kaksoisnapsauttamalla lataamaasi tiedostoa.

Ennen asennuksen alkua voit valita asennuksen aikana käytettävän kielen. Seuraavassa ikkunassa voit valita OmegaT-ohjelman käyttämän kielen. Kun rastitat haluamasi kielen, niin OmegaT.l4J.ini-tiedosto asettuu käyttämään tätä kieltä (lisätietoja tästä löydät seuraavasta osiosta). Kun olet hyväksynyt käyttöoikeussopimuksen, asennusohjelma kysyy, haluatko luoda ohjelmalle kansion Käynnistys -valikkoon ja haluatko luoda työpöydälle ohjelman pikakuvakkeen sekä lisätä ohjelman pikakäynnistyspalkkiin. Voit luoda nämä oikotiet myöhemmin vetämällä OmegaT.exe-tiedoston linkiksi työpöydälle tai Käynnistys-valikkoon. Viimeisessä ikkunassa voit käydä läpi asentamasti version lueminut-tiedoston ja ohjelman asennuksen tekemät muutokset tiedostoihin.

OmegaT-ohjelman ajaminen

Kun OmegaT on asennettu koneelle, voit käynnistää ohjelman joko napsauttamalla tiedostoa OmegaT.jar tai käyttämällä komentoriviä.

Yksinkertaisin tapa käynnistää ohjelma on kuitenkin suorittaa OmegaT.exe-ohjelma. Ohjelman käynnistysvaihtoehdot luetaan tällöin OmegaT.l4J.ini-tiedostosta, joka on samassa kansiossa kuin .exe-tiedosto. Voit muokata .ini-tiedostoa koneesi asetusten mukaiseksi. Seuraavan esimerkin mukainen INI-tiedosto vaatii 1 GB:n verran muistia ohjelman käytettäväksi, käyttäjän kieleksi ranskan ja käyttäjän sijainniksi Kanadan:

# OmegaT.exe runtime configuration
# To use a parameter, remove the '#' before the '-'
# Memory
-Xmx1024M
# Language
-Duser.language=FR
# Country
-Duser.country=CA

Suositus: Jos OmegaT toimii hitaasti Windowsin etätyöpöytäistuntojen aikana, voit käyttää tätä valintaa:

-Dsun.java2d.noddraw=false

OmegaT:n päivittäminen

Nämä tiedot koskevat ainoastaan "perinteisiä" OmegaT:n Windows-versioita. Ne eivät koske automaattisesti päivittyviä Web Start -versioita eivätkä Windowsiin asennettuja käyttöympäristöjen välisiä versioita.

Jos haluat päivittää koneellesi asennetun OmegaT:n uudempaan versioon, käytettävissäsi on kaksi vaihtoehtoa:

  • Olemassa olevan asennuksen päälle asentaminen käy helposti, kun asennuksen alkuvaiheessa valitset uuden version asennettavaksi samaan kansioon, jonne jo olemassa oleva versio asennettiin. Tiedot kirjoitetaan OmegaT:n "vanhan" version päälle, mutta ohjelman kaikki asetukset säilyvät muuttumattomina. Säilyviä asetuksia ovat OmegaT-ohjelmassa tehdyt asetukset, kaikki OmegaT.l4J.ini-tiedostoon tehdyt muutokset sekä ohjelman käynnistävä komentosarja (.bat-tiedosto), mikäli sellainen on käytössä.

Tätä tapaa käyttämällä voit ladata ohjelmasta myös ilman JRE-ympäristöä olevan "Windows without JRE"-version, sillä uusi asennus käyttää koneellasi jo olevaa JRE-ympäristöä.

  • Uuteen kansioon asentaminen mahdollistaa sen, että voit pitää molempia versioita koneellasi rinnakkain esimerkiksi siihen saakka, kunnes olet oppinut käyttämään ohjelman uutta versiota. Myös tämä tapa käyttää OmegaT-ohjelmassa tekemiäsi valintoja ja asetuksia. Huomaa kuitenkin seuraavat seikat:

    • Jos olet tehnyt muutoksia OmegaT.l4J.ini-tiedostoon ja/tai käytät .bat-tiedostoa, sinun on kopioitava nämä uuteen versioon.

    • Jos olemassa oleva OmegaT-asennuksesi on JRE-ympäristön sisältävä "Windows with JRE"-versio, myös uuden version on oltava "Windows without JRE".

Linux (Intel) -käyttäjät

Oikeanlaisen paketin lataaminen

Onko koneellasi Oraclen Java 8 JRE -yhteensopiva Java-ympäristö?

  • Kyllä: Lataa tiedosto OmegaT_4.n.n_Without_JRE.zip.

  • Ei / En tiedä: Lataa tiedosto OmegaT_4.n.n_Linux.tar.bz2.

    Tässä paketissa on mukana Oraclen täydellinen Java-ajoympäristö. Tämä JRE ei vaikuta muihin Java-ympäristöihin, joita koneellesi on mahdollisesti jo asennettu.

OmegaT:n asentaminen

Pura tar-paketti / lataamasi tiedosto. Tämä luo työkansioon omegat/-kansion, josta löydät kaikki OmegaT-ohjelman ajamiseen tarvittavat tiedostot. Pura .tar.gz-tiedosto tällä komennolla:

$ tar xf downloaded_file.tar.gz

OmegaT:n lisääminen valikkoihin (KDE) tai paneeleihin (Gnome)

KDE 4 -käyttäjät

Näin lisäät OmegaT:n valikkoihisi:

  • Paina Alt+F2 , niin saat KRunnerin näkyviin. Kirjoita kmenuedit ja paina enter-näppäintä, niin suoritat komennon. Näytölle avautuu KMenuEditor-ikkuna. Valitse KMenuEditor-ikkunassa File > New Item .

  • Valitse sopiva valikko ja lisää sinne alivalikko tai valikon kohta valitsemalla File – New Submenu tai File – New Item . Kirjoita uuden kohteen nimeksi OmegaT.

  • Siirry Command-kenttään ja etsi siirtymispainikkeen avulla OmegaT-ohjelman käynnistävä komentosarja (puretussa kansiossa oleva OmegaT-niminen tiedosto) ja valitse se.

  • Napsauta ikonipainiketta (Name/Description/Comment-kenttien oikealla puolella).

  • Valitse Other Icons – Browse, ja siirry OmegaT-sovelluskansion /images-alikansioon. Valitse OmegaT.png-ikoni.

  • Lopuksi tallenna tekemäsi muutokset valitsemalla File – Save .

GNOME-käyttäjät

Näin lisäät OmegaT:n valikkoihisi:

  • Napsauta hiiren kakkospainikkeella paneelia Add New Launcher .

  • Kirjoita Name-kenttään OmegaT. Siirry Command-kenttään ja etsi siirtymispainikkeen avulla OmegaT-ohjelman käynnistävä komentosarja (puretussa kansiossa oleva OmegaT-niminen tiedosto) ja valitse se. Vanhvista valinta napsauttamalla OK.

  • Napsauta ensin ikonipainiketta, sitten Browse… ja siirry OmegaT-sovelluskansion /images-alikansioon. Valitse OmegaT.png-ikoni. GNOME ei välttämättä pysty näyttämään käytettävissä olevia ikonitiedostoja, sillä se odottaa ikonitiedostojen olevan SVG-tiedostoja. Jos kuitenkin valitset tämän kansion, tiedostojen pitäisi tulla näkyviin ja voit valita OmegaT-png-tiedoston ohjelman ikoniksi.

OmegaT-ohjelman ajaminen

Voit käynnistää OmegaT-ohjelman joko komentoriviltä käyttämällä komentosarjaa määritteineen tai napsauttamalla OmegaT.jar-tiedostoa. Käynnistystavat vaihtelevat jakelusta riippuen. Varmista, että PATH-asetuksesi ovat oikein ja että .jar-tiedostot on liitety oikein Java Launcher -käynnistysohjelmaan. Lisää tietoa löydät alhaalta kohdasta "OmegaT:n käynnistäminen komentoriviltä".

Macin käyttäjät

Oikeanlaisen paketin lataaminen

OmegaT:ssä on mukana Java JRE 1.8.

Lataa tiedosto OmegaT_4.n.n_Mac.zip.

OmegaT:n asentaminen

Pura tiedosto OmegaT_4.n.n_Mac.zip kaksoisnapsauttamalla sitä. Tällöin koneelle muodostuu kansio OmegaT. Kansiossa on kaksi tiedostoa, index.html ja OmegaT.app. Kopioi kansio sopivaan kansioon (esim. Ohjelmat). Kun olet tehnyt tämän, voit poistaa OmegaT_4.n.n_Mac.zip-tiedoston, sillä sitä ei enää tarvita.

OmegaT:n lisääminen Dockiin.

Vedä ja pudota OmegaT.app-tiedosto Dockiin.

OmegaT-ohjelman ajaminen

Kaksoisnapsauta OmegaT.app-tiedostoa tai napsauta sitä Dockissa.

Voit muokata OmegaT:n käyttäytymistä sekä Asetukset -kohdassa että muokkaamalla paketin Configuration.properties-tietoa.

Pääset käsiksi Configuration.properties-tiedostoon napsauttamalla OmegaT.app-tiedostoa hiiren kakkospainikkeella ja valitsemalla "Näytä pakkauksen sisältö" (Show Package Contents). Tiedosto on kansiossa Contents/Resources. Napsauta sitä hiiren kakkospainikkeella ja avaa se haluamassasi tekstieditorissa. Voit myös käyttää komentorivillä komentoa cd suoraan kansioon ja avata Configuration.properties-tiedoston komentorivipohjaisessa editorissa kuten emacs tai vi.

Muutat ohjelman asetuksia muokkaamalla tiedostoa Configuration.properties. Tiedostossa olevat ennalta määritetyt asetukset saat käyttöön poistamalla ennen parametriä olevan #-merkin. Esimerkiksi parametri user.language=ja (ilman #-merkkiä) käynnistää OmegaT:n japaninkielisen käyttöliittymän.

Saat muutettua OmegaT:n käytössä olevaa muistia muokkaamalla tiedostoa OmegaT.app/Contents/Info.plist : Poista rivin <!-- <string>-Xmx6g</string> --> kommenttimerkit <!-- ja -->, jolloin OmegaT saa käyttöönsä 6 GB muistia. Voit muuttaa komennon kohdan 6g haluamaksesi.

Jos haluat käynnistää useita OmegaT.app-esiintymiä, kaksoisnapsauta kansiossa OmegaT.app/Contents/MacOS/ olevaa OmegaT -tiedostoa.

Käynnistä OmegaT komentoriviltä käyttämällä kansiossa OmegaT.app/Contents/MacOS/Java/ olevaa OmegaT.jar-tiedostoa. Lisää tietoa löydät alhaalta kohdasta "OmegaT:n käynnistäminen komentoriviltä".

macOS-herkut

OmegaT.app voi käyttää macOS-palveluita. Näin voit valita sanan missä päin OmegaT-ohjelmaa tahansa ja käyttää macOS-palveluita tarkistamaan tämän sanan esim. Spotlightin tai Googlen avulla. Voit myös käyttää AppleScriptiä tai Automatoria luomaan palveluja tai komentosarjoja, jotka automatisoivat usein käyttämäsi toiminnot.

Muut järjestelmät

Nämä tiedot koskevat järjestelmiä kuten Solaris SPARC/x86/x64, Linux x64/PowerPC, Windows x64.

Oikeanlaisen paketin lataaminen

OmegaT-paketti on saatavilla Oracle Java JRE:n kanssa sekä Linux (Intel x86)- että Windows-käyttöjärjestelmiin. Jotta OmegaT:tä voi käyttää muissa käyttöjärjestelmissä (Linux PowerPC, Linux x64, Solaris SPARC/x86/x64, Windows x64 jne.), niissä on oltava yhteensopiva Java JRE-ympäristö.

Onko koneellasi Oraclen Java 8 JRE -yhteensopiva Java-ympäristö?

  • Kyllä: Lataa tiedosto OmegaT_4.n.n_Windows_without_JRE.zip. Tätä pakettia voidaan käyttää missä tahansa käyttöjärjestelmässä, johon on asennettu Java 8 -yhteensopiva JRE-ympäristö.

  • En tiedä: Avaa pääteohjelma ja kirjoita komento java -version . Jos vastaus on "command not found" tai sen kaltainen, Javaa ei todennäköisesti ole asennettu järjestelmääsi.

  • Ei: Hanki järjestelmääsi sopiva Java JRE -ympäristö (ks. alla) ja lataa tiedosto OmegaT_4.n.n_Without_JRE.zip.

    Oracle tarjoaa JRE-ympäristöjä Solaris SPARC/x86 (Java 8)-, Linux x64-, Solaris x64- sekä Windows x64 (Java 8)- käyttöjärjestelmiin osoitteessa http://www.oracle.com/technetwork/java/archive-139210.html.

    IBM tarjoaa JRE-ympäristöjä Linux PowerPC -käyttöjärjestelmään osoitteessa http://www.ibm.com/developerworks/java/jdk/linux/download.html.

    Seuraa tarvitsemasi pakentin asennusohjeita.

OmegaT:n asentaminen

Asenna OmegaT yksinkertaisesti purkamalla tiedosto OmegaT_4.n.n_Without_JRE.zip. Tämä luo työkansioon ./OmegaT_4.n.n_Without_JRE/-kansion, josta löydät kaikki OmegaT-ohjelman ajamiseen tarvittavat tiedostot.

OmegaT-ohjelman ajaminen

Kun OmegaT on asennettu, voit käynnistää ohjelman suoraan komentoriviltä, voit luoda komentoriville komentosarjan määritteineen tai voit avata ohjelman suoraan napsauttamalla OmegaT.jar-tiedostoa. Käynnistystavat vaihtelevat jakelusta riippuen. Varmista, että PATH-asetuksesi ovat oikein ja että .jar-tiedostot on liitety oikein Java Launcher -käynnistysohjelmaan. Tarkista alla oleva "<l3/>" saadaksesi lisää tietoa.

Vedä ja pudota

Useimmissa järjestelmissä OmegaT-projektin voi avata vetämällä ja pudottamalla omegat.project-tiedoston työpöydällä olevaan OmegaT-ikoniin. Projekti voi olla mahdollista avata myös kaksoisnapsauttamalla omegat.project-tiedostoa.

Java Web Startin käyttö

Java Web Start -teknologia (osa Java 8:aa ja seuraavia) mahdollistaa sen, että erilliset Java-sovellukset voidaan ottaa käyttöön yhdellä napsautuksella tietoverkon kautta. Java Web Start varmistaa, että käyttöön otetaan sekä sovelluksen uusin versio että oikea Java Runtime Environment (JRE). Kun käynnistät OmegaT:n ensimmäistä kertaa Java Web Startin avulla, kirjoita seuraava URL selaimesi osoiteriville:

https://omegat.sourceforge.io/webstart/OmegaT.jnlp

Lataa tiedosto OmegaT.jnlp ja napsauta sitä. Asennuksen aikana saatat saada useita turvallisuusvaroituksia, käyttöjärjestelmäsi riippuen. Oikeudet, jotka annat Web Start -versiolle (esim. "rajoittamaton tietokoneen käyttöoikeus"), ovat täsmälleen samat kuin paikallisella versiolla, eli versio voi käyttää tietokoneen kovalevyä. Seuraavat OmegaT.jnlp-tiedoston napsautukset tarkistavat mahdolliset päivitykset, asentavat ne ja lopulta käynnistävät OmegaT:n. Ensimmäisen asennuksen jälkeen voit tietysti käyttää OmegaT.jnlp-ohjelmaa myös offline-tilassa.

Tietosuoja : OmegaT Java Web Start ei tallenna mitään tietojasi muualle kuin tietokoneelle, jossa käytät ohjelmaa. Sovellus on käynnissä ainostaan omalla koneellasi. Käyttämäsi asiakirjat ja käännösmuistit pysyvät omalla koneellasi, eikä OmegaT-projektilla ole pääsyä työhösi tai tietoihisi.

Mikäli haluat tai sinun on pakko käyttää käynnistyskomennon argumentteja (ks. yllä), huomaa, että sinun on käytettävä tavallista asennustapaa.

OmegaT:n käynnistäminen komentoriviltä

Tavallisesti OmegaT:tä ei tarvitse käynnistää komentoriviltä. Komentorivivaihtoehto antaa kuitenkin käyttäjän ohjata ja muokata ohjelman käyttäytymistä. OmegaT:n voi käynnistää komentoriviltä kahdella tavalla:

Komentoikkunan avaaminen

Komentoikkunaa kutsutaan myös pääteikkunaksi. Windowsissa sen nimi on MS-DOS-ikkuna. Löydät sen Käynnistys-valikosta.ä Vastaava sovellus macOS-käyttöjärjestelmässä on Pääte, jonka löydät kohdasta Ohjelmat > Lisäohjelmat.

Sinun on tavallisesti kirjoitettava kaksi komentoa käynnistääksesi OmegaT:n Ensimmäinen komento on

cd 
                     kansio

jossa kansio on sen kansion koko polku, jonne olet tallentanut OmegaT ohjelman, eli tarkemmin sanottuna OmegaT.jar-tiedoston sijainti. Käytännössä tämä komento näyttää kuta kuinkin tältä:

Windows-koneissa

cd C:\Program Files\OmegaT

Macissa

cd 
                     <OmegaT.app-sijainti>
                  /OmegaT.app/Contents/Resources/Java/

Linux-koneissa

cd /usr/local/omegat

Tämä komento vaihtaa kansioksi suoritettavan OmegaT-tiedoston sisältävän kansion. Toinen komento käynnistää OmegaT:n. Perusmuotoisimmillaan tämä komento on

java -jar OmegaT.jar

Huomaa isot ja pienet kirjaimet: Muissa kuin Windows-käyttöjärjestelmissä ohjelma ei käynnisty, jos kirjoitat omegat etkä OmegaT!

Tämän käynnistystavan etuna on se, että se paljastaa ongelmien syitä. Jos ohjelman käytön aikana tapahtuu virhe, pääteikkunaan tulee virheilmoitus, jossa voi olla hyödyllistä tietoa virheen syystä.

Yllä kuvattu tapa on hieman epäkäytännöllinen, kun ohjelma käynnistetään tämän tästä. Tämän vuoksi yllä kuvatut komennot ovat tiedostossa ("komentojonossa", Windows-käyttöjärjestelmän .bat-tiedostossa).

Kun BAT-tiedosto suoritetaan, sen sisältämät komennot suoritetaan automaattisesti. Jos siis haluat muuttaa käynnistyskomentoa, riittää, että muokkaat tätä tiedostoa.

Käynnistyskomennon argumentit

Peruskomento on jo mainittu edellä. Komentoa muutetaan lisäämällä siihen argumentteja. Argumentit lisätään aloittavan java -komennon jälken, ennen komentoa -jar OmegaT.jar. Huomaa, että Windowsissa voit muuttaa OmegaT.l4J.ini-tiedostoa mieleiseksesi. Saman voit tehdä muokkaamalla macOS-käyttöjärjestelmässä Configuration.properties-tiedostoa tai Linux-käyttöjärjestelmässä OmegaT-käynnistysohjelmaa.

Alla on luettelo käytössä olevista argumenteista. Pääteikkunan komento man java antaa kokeneille käyttäjille lisätietoa argumenteista.

  • Käyttöliittymän kieli

    -Duser.language= XX Tavallisesti, eli kun OmegaT käynnistetään ilman argumentteja, ohjelma tunnistaa ensiksi käyttäjän käyttöjärjestelmän kielen. Jos käyttöliittymä tällä kielellä on olemassa, OmegaT käyttää sitä. Jos siis käyttäjän käyttöliittymä on venäjäksi ja OmegaT on lokalisoitu venäjäksi, OmegaT-käyttöliittymä valikkoineen kaikkineen on venäjäksi. Jos käyttöjärjestelmän kieltä ei ole saatavilla, OmegaT käyttää oletusarvoisesti englantia. Tämä on vakiokäytäntö.

    Argumentti -Duser.language= XX saa OmegaT:n käyttämään käyttöjärjestelmän kielen sijasta määriteltyä kieltä. Kommennon XX tarkoittaa halutun kielen kaksimerkkistä kielikoodia. Ranskankielisen OmegaT:n käynnistämiseen (esim. venäjänkielisessä käyttöliittymässä) tarvittava komento on siis

    java -Duser.language=fr -jar OmegaT.jar

  • Käyttäjän maa

    -Duser.country= XX Kielen lisäksi voit määritellä myös maan, esimerkiksi CN tai TW kiinan kielelle. Jotta pika-aloitusopas näkyisi halutulla kielellä, sinun on määriteltävä sekä kieli että maa. Tämä on pakollista, vaikka kielen ja maan yhdistelmiä olisi vain yksi, kuten esimerkiksi pt_BR, brasilianportugali.

  • Käytettävissä olevan muistin määritys

    -Xmx ?? M Tämä komento antaa OmegaT:n käyttöön lisää muistia. Oletusarvoisesti muistia on käytettävissä 1024 MB, joten tätä vähemmän muistia ei ohjelmalle kannata määrittää. ?? tarkoittaa määritellyn muistin määrää megatavuina. Komento, joka käynnistää OmegaT:n ja antaa ohjelman käyttöön 2048 MB (2 GB) muistia, on

    java -Xmx2048M -jar OmegaT.jar

  • Välityspalvelimen IP-osoite

    -Dhttp.proxyHost= nnn.nnn.nnn.nnn Järjestelmäsi mahdollisesti käyttämän välityspalvelimen IP-osoite.

  • Välityspalvelimen portin numero

    -Dhttp.proxyPort= NNNN Portin numero, jota järjestelmäsi käyttää välityspalvelinyhteyteen.

  • Google Translate V2

    -Dgoogle.api.key= A123456789B123456789C123456789D12345678 Jos olet ottanut käyttöön Googlen käännöspalvelut, kirjoita tähän yksityinen Google API-avaimesi. Huomaa, että avain on 38 merkkiä pitkä.

  • Microsoft Translator

    Varmista, että sinulla on ilmainen Microsoft-tili. Tarvitset tätä Windows Azure Marketplace -kauppapaikalle kirjautumiseen ja Translator-palvelun käyttöön. Huomaa, että voit hakea kuukaudessa 2 000 merkin verran ilmaiseksi. Löydät kaksi tarvittavaa merkintää tilisivullasi kohdista Ensisijainen tiliavain (Primary account key) sekä Asiakas-ID (Customer-ID)

    -Dmicrosoft.api.client_id=
                              XXXXXXXX-XXXX-XXXX-XXXX-XXXXXXXXXXXX
                           
                        
    -Dmicrosoft.api.client_secret=
                              XXXX9xXxX9xXXxxXXX9xxX99xXXXX9xx9XXxXxXXXXX=

  • Yandex Translate

    Varmista, että sinulla on ilmainen Yandex-tili. Tarvitset sitä saadaksesi Yandex Translate API-avaimen ja käyttääksesi sitä. Voit pyytää API-avainta lomakkeella API key request form, ja näet avaimen My Keys -sivulla.

    -Dyandex.api.key=
                              trnsl.1.1.XXXXXXXXXXXXXXXX.XXXXXXXXXXXXXXXX.XXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX

Argumentteja voi yhdistää. Komento, joka käynnistää OmegaT:n kaikilla yllä mainituilla argumenteilla, olisi:

java -Dswing.aatext=true -Duser.language=pt -Duser.country=BR -Xmx2048M -Dhttp.proxyHost=192.168.1.1 -Dhttp.proxyport=3128 -jar -OmegaT.jar

Komentorivimuotoinen OmegaT

Konsolimuotoista OmegaT-ohjelmaa käytetään komentosarjaympäristössä. Kun ohjelma käynnistetään konsolimuodossa, graafista käyttöliittymää ei avata (eli ohjelma toimii missä tahansa konsolissa), ja käännösprojektia käsitellään automaattisesti annettujen komentojen mukaan.

Edellytykset

OmegaT:tä voi ajaa komentorivitilassa vain, kun on olemassa kelvollinen OmegaT-projekti. Projektin sijainnilla ei ole väliä, sillä sinun on joka tapauksessa lisättävä se komentoriville ohjelman käynnistyksen yhteydessä.

Määritystiedostojen on oltava saatavilla, mikäli tarvitset muutettuja asetuksia. Voit tehdä tämän kahdella tavalla:

  • Käytä OmegaT:tä normaalisti (graafisen käyttöliittymän kautta) ja määrittele asetukset. Jos käynnistät OmegaT:n konsolitilassa, ohjelma käyttää samoja asetuksia.

  • Jos et voi käyttää OmegaT:tä normaalisti (graafista käyttöliittymää ei ole saatavilla), kopioi toiselle koneelle asennetun OmegaT:n asetustiedostot tiettyyn kansioon omalle koneellesi. Tiedostosijainnilla ei ole väliä, sillä voit lisätä sen komentoriville ohjelman käynnistyksen yhteydessä. Tärkeät tiedostot ovat filters.conf ja segmentation.conf. Löydät ne käyttäjän kotikansiosta (esim. Windowsissa C:\Documents and Settings\ user \OmegaT tai Linuxissa ~/.omegat/).

Käynnistys konsolitilassa

OmegaT:n käynnistäminen konsolitilassa vaatii muutamia ylimääräisiä parametrejä. Tärkein niistä on /koko/projektin/projektipolku sekä valinnaisesti --config-dir= /yksittaisten/maaritystiedostojen/polku/ . Esimerkki:

java -jar OmegaT.jar 
                        /koko/projektin/projektipolku
                      \
    --config-dir=
                        /yksittaisten/maaritystiedostojen/polku/
                      \
    --config-file=
                        /yksittaisten/maaritystiedostojen/polku/
                      \
    --mode=
                        console-translate|console-createpseudotranslatetmx|console-align
                      \
    --source-pattern=
                        regexp

Huomioi, että kaikkien parametrien alussa on kaksi - merkkiä.

Selitys:

  • /koko/projektin/projektipolku kertoo OmegaT:lle, mistä se löytää käännettävän projektin. Jos polku on annettu, OmegaT käynnistyy konsolitilassa ja kääntää kyseisen projektin.

  • --config-dir= /yksittaisten/maaritystiedostojen/polku/ kertoo OmegaT:lle, mihin kansioon määritystiedostot on tallennettu. Jos polkua ei ole annettu, OmegaT käyttää oletusasetuksia (käyttäjän kotikansion alla olevaa OmegaT-kansiota, tai jos sitä ei ole, kyseistä työskentelykansiota). Huomaa kaksinkertaiset - merkit.

  • --config-file= /yksittaisten/maaritystiedostojen/polku// kertoo OmegaT:lle, mitä määritystiedostoa sen tulaa käyttää.

  • --mode= … OmegaT käynnistyy konsolitilassa ja suorittaa yhden seuraavista palveluista automaattisesti:

    • --mode=console-translate

      Tässä tilassa OmegaT yrittää kääntää kansiossa /source/ olevat tiedostot saatavilla olevien käännösmuistien avulla. Tämä on hyödyllistä silloin, kun OmegaT:tä käytetään palvelimella, jonne projektin TMX-tiedostot syötetään automaattisesti.

    • --mode=console-createpseudotranslatetmx

      Tässä tilassa OmegaT luo koko projektille ainoastaan lähdetiedostoihin perustuvan käännösmuistin. Määrittele luotava TMX-tiedosto komennolla

      --pseudotranslatetmx= kaikkisegmentit.tmx --pseudotranslatetype= equal|empty

      Argumentti pseudotranslatetype määrittelee, ovatko kohdesegmentit samoja kuin lähdesegmentit, vai jätetäänkö kohdesegmentit tyhjiksi.

    • --mode=console-align

      Tässä tilassa OmegaT kohdistaa projektin /source/-kansiossa olevat tiedostot ja annetussa sijainnissa olevan sisällön. Tästä saatava TMX-tiedosto tallentuu /omegat/-kansioon nimellä align.tmx. Se, mitä tiedostoja kohdennus käyttää, riippuu siitä, tuetaanko niiden tiedostosuodatinta. Joitakin tuettuja suodattimia: ILIAS Language File, Java(TM) Resource Bundles, Key=Value Text, Magento CE Locale CSV, MoodlePHP, Mozilla DTD, Mozilla FTL, PO, RC, SubRip Subtitles, Windows Resources

      Tässä tapauksessa tarvitaan yksi lisäparametri, joka määrittelee kohdedatan sijainnin:

      --alignDir= <käännettyjen tiedostojen sijainti>

      Komennossa alignDir on oltava projektin kohdekielellä oleva käännös. Jos projekti on esimerkiksi englannista ranskaan, alignDir-komennossa on oltava _fr-päätteinen nippu. Tästä saatava TMX-tiedosto tallentuu /omegat/-kansioon nimellä align.tmx.

  • --source-pattern= regexp

    Tämä valinta määrittelee tiedostot, jotka käsitellään automaattisesti sen jälkeen, kun tilaa on käytetty. Jos parametriä ei ole määritelty, kaikki tiedostot käsitellään. Tässä pari esimerkkiä tavallisista tiedostorajoituksista

    • .*\.html

      Kaikki HTML-tiedostot käännetään. Huomaa, että tavallisessa ilmauksessa *.html käytetty piste täytyy säännöllisessä lausekkeessa ilmaista pakomerkin avulla (\.).

    • test\.html

      Ainoastaan lähdekansion juuressa oleva test.html-tiedosto käännetään. Muissa kansioissa olevat test.html-tiedostot jätetään huomiotta.

    • dir-10\\test\.html

      Ainoastaan dir-10kansiossa oleva test.html -tiedosto käsitellään. Huomaathan, että myös kenoviivamerkki täytyy ilmoittaa pakomerkin avulla.

  • --tag-validation= abort|warn tuotetunTiedostonNimi

    Tämä valinta mahdollistaa erätilassa tapahtuvan tägien tarkistuksen. Jos valintana on abort, tägien tarkistus pysähtyy ensimmäisen epäkelpoisen segmentin kohdalle. Jos valintana on warn, tägien tarkistus käsittelee kaikki segmentit ja kirjoittaa varoitukset epäkelpoisia tägejä sisältävistä segmenteistä määriteltyyn tiedostoon.

  • --no-team koskee ryhmätyöprojekteja. Käytä tätä, jos OmegaT:n ei tule synkronoida projektin sisältöä.

  • --disable-project-locking sallii Windows-käyttöjärjestelmässä sen, että sama projekti avataan useissa OmegaT-istunnoissa. Windowsissa tiedosto omegat.project on oletusarvoisesti lukittu, jolloin käyttäjä saa virheilmoituksen, jos hän yrittää avata projektia, joka on jo avoinna toisessa OmegaT-istunnossa. Kun tämä parametri on käytössä, tiedostoa ei lukita.

Hiljainen valimta

Konsolitilan komennoissa voidaan käyttää ylimääräistä parametriä --quiet. Hiljaisessa tilassa ruudulle kirjataan tavallista vähemmän tietoa. Tavallisesti tilarivillä näkemiäsi viestejä ei näytetä.

Käyttö: java -jar OmegaT.jar /koko/projektin/projektipolku --mode=console-translate --quiet

OmegaT:n rakentaminen lähdekoodista

Löydät nykyisen version lähteet joko tietovarastosta Git-asiakkaan avulla https://git.code.sf.net/p/omegat/code tai suoraan SourceForgesta.

Kun olet ladannut lähdekoodin, avaa lähdekansio komentorivillä ja kirjoita

gradlew assembleDist

Komento luo täyden OmegaT-jakelun ./jakelu/versiot-kansioon. Löydät sieltä zip-kansion, jossa on kaikki, mitä tarvitset OmegaT:n käyttöön.

Voit myös suorittaa ohjelman suoraan komennolla

gradlew run

Yksityiskohtaiset tiedot rakentamisesta löydät docs_devel readme-tekstistä.
