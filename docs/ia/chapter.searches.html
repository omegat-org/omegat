Fenestra Recerca

Aperi le Fenestra de recerca con Ctrl + F e insere le parola o le phrase tu pro le qual tu desira cercar in le quadro Cercar pro .

In alternativa, tu pote eliger un parola o un phrase in le quadro del Editor, quadro del Concordantias partial o quadro del Glossario e pulsar Ctrl + F . Le parola o phrase es inserite automaticamente in le quadro Cercar pro Tu pote haber plure Fenestras de recerca aperte al mesme tempore, sed claude los quando illos non son son plus necessari a fin que illos non incombra tu scriptorio.

Clicca le flecha a cadita del cassa Recercar pro al fin de acceder al ultime 10 recercas.

Le fenestra de Recerca ha su proprie menus:

  • File > Recercar pro le selection ( Ctrl + F ): re-focalisa sur le campo de recerca e elige tote su contentos.

  • File > Clauder ( Ctrl + W ): claude le fenestra de recerca (in le mesme maniera de Esc )

  • Rediger > Inserer fonte ( Ctrl + Shift + I ): insere le segmento fonte actual.

  • Rediger > Replaciar con le fonte ( Ctrl + Shift + R ): replacia con le actual segmento fonte.

  • Rediger > Crear un entrata de glossario ( Ctrl + Shift + G ): adde un nove elemento al glossario.

Usar characteres universal

In ambe le recercas exacte e per parola clave, on pote usar le characteres de recerca universal '*' e '?'. Illos ha le significato, familiar al usatores de Word:

  • '*' concorda zero o plus characteres, ab le position actual in un date parola al su fin. Le termino de recerca 'run*' per exemplo facerea concordar le parolas 'run', 'runs' e 'running'.

  • '?' equala omne singule character. Per exemplo, 'run?' facerea concordar le parola 'runs' e 'runn' in le parola 'running'.

Le concordantias essera marcate in azur hardite. Nota que '*' e '?' ha significato special in le expressiones regular, assi le recerca con carta salvage, ut describite ci, refere se al exacte e recerca parola clave solmente (vide in sequito).

Methodos de recerca e optiones

Selige le methodo per le buttones radio. Son disponibile le methodos de recerca sequente:

Cerca &Exacte

Recerca pro le segmentos continente le linea de character exacte que tu ha specificate. Un recerca exacte quere pro un phrase, id es si plure parolas son entrate, illos son trovate solmente si illos occurre in justo celle sequentia. Usante le recerca per parola clave con open file trovara assi omne occurrentias del linea de character open file ma non file opened o open input file .

recerca parola clave

Cerca pro le segmentos continente tote le parolas clave tu specificava, in omne positura. Selige "recerca parola clave" per perquirer pro omne numero de individual parolas complete, in omne ordine. OmegaT monstra un lista de omne segmentos continente tote le parolas specificate. Recercas per parola clave es simile a un recerca "con tote le parolas" in un motor de recerca Internet como Google (AND logic). Usante le recerca per parola clave con open file trovara assi omne occurrentias del linea de character open file, assi como file opened, open input file, file may not be safe to open , et cetera.

expressiones regular

Le linea de character de recerca essera tractate como un expression regular. Le linea de character de recerca - [a-zA-Z]+[öäüqwß] - in le caso precedente per exemplo appare pro parolas in le segmento de destination, continente characteres dubitabile del claviero german. Le expressiones regular son un maniera potente pro cercar pro occurrentias de un linea de character. Vide plus in le capitulo Expressiones regular.

In addition pro uno del methodos precedente tu pote seliger le sequente:

  • sensitive al majusculas e minusculas : le recerca essera effectuate pro le exacte linea de characteres specificate; id es es observate le capitalisation.

  • Concordantias del spatio nbsp : quando es marcate iste option, un character spatio del fila de recerca concorda sia un normal character de spatio que un character (\u00A) de spatio ininterrumpibile.

  • in le fonte: cerca in le segmentos fonte

  • in le traductiones: cerca in le segmentos final

  • in le notas: cerca in le notas del segmentos

  • in le commentos: cerca in le commentos del segmentos

  • Rendite o non rendite: recerca in ambe le segmentos rendite e non rendite.

  • Rendite: recerca sol in le segmentos rendite.

  • Non rendite: recerca sol in le segmentos non rendite.

  • Monstrar: tote le segmentos concordante: si marcate, tote le segmentos es monstrate uno per uno, etsi si illos es presente plure vices in le mesme documento o in documentos differente.

  • Monstrar: le nomines de file: si marcate, le nomine de file ubi cata segmento es trovate, es monstrate ante cata resultato.

  • Cercar in le projecto : marcar Memoria pro includer le memoria de traduction del projecto (le file project_save.tmx) in le recerca. Marcar TMs pro includer in le recerca le memorias de traduction ubicate in le plica tm. Marcar Glossarios pro includer in le recerca le glossarios ubicate in le plica glossary.

  • Recerca in le files: recerca in un file singule o in un plica continente un collection de files. Quando tu cerca trans le files (al contrario que al memorias de traduction), OmegaT restringe le recerca al files in le formatos del file fonte. Per consequentia, ben que OmegaT is bastante capace a tractar files tmx, illo non include los in le Files a perquirer del recerca.

Si tu clicca sur le button Optiones avantiate pote esser selecte altere criterios (autor o cambiator del traduction, date del traduction, exclusion de segmentos orphane, et cetera). Quando es marcate le option insensibile al largessa del character , le recercas pro le formas plen (characteres CJK) concordara le formas a largessa medietate e vice versa.

Monstra le resultatos del recerca

Pulsante le button de recerca, post inscribite in le campo un linea de character, monstra tote le segmentos in le projecto que include le linea de character entrate. Pois que OmegaT tracta segmentos identic como un entitate singule, sol le prime segmento unic es monstrate. Le segmentos son monstrate in ordine de apparition in le projecto. Le segmentos rendite son monstrate con le texto original al alto e le texto rendite al basso, le segmentos non rendite son monstrate con le fonte sol.

Le duple-clic sur un segmento aperi lo in Editor pro modificationes (un clic singule face lo quando es marcate le option Auto-synchronisar con le Editor ). Tu pote alora cambiar a retro al Fenestra de Recerca pro trovar le segmento successive, per exemplo pro controlar e, si necesse, corriger le terminologia.

In le fenestra de Recerca, tu pote usar le vias breve standard ( Ctrl + N , Ctrl + P ) pro mover ab una segmento a un altere.

Tu pote haber plure Fenestras de Recerca aperte al mesme tempore. Tu pote rapidemente vider lor contentos querente a lor titulo: il continera le termino de recerca usate.

Entratas filtro in editor conforme al recerca

Pro plus facile navigation in le recerca resultato imponite, tu pote applicar le recerca al editor. Pulsa le button Filtro sur le fundo pro limitar le entratas monstrate in le fenestra Modifica pro illos que concorda le recerca currente. Tu pote usar le navigation normal pro adir per exemplo le segmento (non rendite) successive que concorda le criterios de recerca.

NB:

  • Un recerca pote esser limitate a 1000 articulos, assi si tu recerca sur un phrase commun, le editor monstra solmente illes 1000 entratas concordante, e non tote les entratas que concorda le criterios de recerca.
  • Un file pote haber nulle entratas concordante, assi illo monstrara se vacue.
  • Si un recerca elimina le duplicatos, ille duplicatos non essera in le Editor.

Pro remover un filtro, pulsa le button Remove filtro , o recarga le projecto.
