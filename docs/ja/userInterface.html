ユーザーインターフェースの項目

OmegaT のユーザーインターフェースは多くのアイテムからなります。以下ではまずそれらを要約し、その後でそれぞれについてより詳細について記述します。

最もよく使うウィンドウ

  • OmegaT メインウィンドウ は 3 つのペインからなります。
    • 翻訳を入力、修正するエディタペイン
    • 翻訳メモリから最も似ている分節を表示する参考訳文ペイン
    • 用語集を表示する用語集ペイン
    • OmegaT のほとんどの機能にアクセスできる メインメニュー
    • ウィンドウの下部にあり、作業に関連するメッセージが表示されるステータスバー
  • 翻訳するファイルの一覧が表示される翻訳対象ファイル一覧ウィンドウ
  • プロジェクトの特定の分節を見つけるために使われる文字列検索ウィンドウ

時々使うウィンドウ

  • 訳文文書のフォーマットの間違いを検出するためのタグ検証ウィンドウ
  • 取扱説明ガイドを読むためのヘルプブラウザ

プロジェクトの設定を変えるときに使うダイアログ

  • プロジェクトフォルダと言語を変更するためのプロジェクト設定
  • OmegaT が原文、訳文、参考訳文、用語を表示するのに使うフォントを修正するためのフォント設定
  • 対応しているファイル形式の扱いを調整するためのファイルフィルタ
  • 文章を文に分節化する方法を変更するための分節化
  • 分節を移動するときの OmegaT の振る舞いを変更するための翻訳入力行

OmegaT メインウィンドウ

メインウィンドウは 3 つのペインとメインメニューとステータスバーからなります。

ペインの位置は変更できますし、ペインをドラッグすることで別のウィンドウに分離することもできます。ペインの状態によって、ペインの右上に 5 つの異なった表示が現れます。表示の上にマウスポインタを置くことで、ツールチップが表示されます。

ステータスバーの近くに、名前だけの形にペインを縮小します。
そのペインが利用可能なウィンドウスペースの全部を使うようにします。
ペインを、最小化または最大化する前の位置に戻します。
ペインをメインウィンドウから切り離して、独立したウィンドウにします。
そのペインをメインウィンドウの中に戻します。

ペインを重ね合わせることも可能です。この場合、ペインは上部のタブとして表示されます。タブをクリックすることでそのペインが前面に表示されます。ペインの間のセパレータをドラッグすれば、ペインのサイズを変えられます。

エディタペイン

ここで翻訳を入力、修正します。エディタペインには部分的に翻訳された文章が表示されます:既に翻訳された文章は翻訳されて表示され、まだ翻訳されていない文章は原文言語で表示されます。

表示されている文章は分節に分割されており、文書をスクロールさせて任意の文節をダブルクリックすることでその分節を修正できます。

分節の一つが有効になっています。一つだけが二つの部分で表示されています:上半分は原文言語で、太字で緑色の背景色でで表示され、下半分は編集フィールドで、二つのマーカーで区切られています:左端のマーカーは"<分節 nnnn>"で、"nnnn"はプロジェクトないの分節番号を表し、右端のマーカーは"</分節>"です。上の部分を参照用に使い、翻訳を含む編集フィールドの内容を上書きまたは修正します。

この文は翻訳されました。この文も翻訳されました。Cette phrase est en train d'être traduite.
<分節 0003>[ここで次のようにタイプすることになります: この文は翻訳中です。]</分節>

移動したときにまだ翻訳されていない編集フィールドは、翻訳入力行の設定によって空白のままとなるか、他の文章が表示されるかします:

  • 原文
  • 翻訳しようとしているものに最も近い文字列の訳文

次の分節に移動すると、訳文は検証されて保存されます。

分節を未翻訳のままにしたい場合は、(Ctrl+Aで全てを選択し、Delで削除することで)単に編集フィールドを空にしてください。

翻訳入力行の設定によっては、OmegaTは訳文が原文と同じでも記憶します。これは、商標や固有名詞など、翻訳されたドキュメントでも同じである必要があるものも含まれている場合に便利です。

詳細については 翻訳文書編集 を参照してください。

参考訳文

参考訳文ペインは、このプロジェクトを翻訳中に作成したものと、過去の作業で作成したり顧客や翻訳業者から受け取ってインポートしたものの 2 種類の翻訳メモリの中から、もっともよく似た分節を表示します。

次の文節に移動したとき、最初の参考訳文(一致率が最も高いもの)が自動的に選択されます。利用可能な参考訳文の数によって、Ctrl+2, 3, 4, 5を押すことで他のものを選択できます。

選択した参考訳文を翻訳に使うには、Ctrl+Rを押して参考訳文で編集フィールドを置き換えるか、Ctrl+Iを押して現在のカーソル位置に挿入します。

大ざっぱに言うと、一致率は比較する分節同士で共通に使われている単語の数を、長い方の分節の単語の数で割ったものです。

選択された参考訳文

  • は太字で表示されます。
  • 翻訳した分節に現れない単語は青色になります。
  • 青色の単語の近くのものや、原文と似ていない位置にあるものは緑色になります。

このような分節を考えます:

   Information about Java & OmegaT

マッチした分節:

   General notes about Java & OmegaT

は以下のように色付けされます:

   General notes about Java & OmegaT

そして割合は 3/5=60%になります。

用語集

用語集ペインは用語集ファイル.から読み込んだ、用語翻訳機能を持ちます。ここに現在の分節で見つかった用語の翻訳を表示しますが、参照のために表示するだけで、用語を翻訳のために挿入したり置換したりすることはできません。OmegaTは複数の単語からなる用語にも対応していますが、非常に基本的な方法を使います: もし用語の全てが現在の分節にあれば、その用語が表示されます。

用語集ペインでの用語のフォーマットは以下のとおりです:

   '原文' = '訳文'
   この用語に関するコメント(あれば)

メインメニュー

明らかに、メインメニューから OmegaT の全ての機能にアクセスできます。もっともよく使う機能はキーボードショートカットから利用可能なので、これに慣れれば翻訳中にメニューを使う必要はなくなります。

全てのメニューとメニュー項目に関する完全な説明についてはメインメニューの項目を参照してください。

ステータスバー

ステータスバーはメインウィンドウの下部にあり、ワークフローに関するメッセージを表示します。このバーは処理中の特定の操作に関するフィードバックメッセージを表示します。また、現在の分節で一致した参考訳文と用語集の数も表示します。

その他のウィンドウ

プロジェクトファイル

翻訳対象ファイル一覧ウィンドウはプロジェクトのファイルの一覧とその他のプロジェクト情報を表示します。これは OmegaT がプロジェクトを読み込んだときに自動的に表示されます。プロジェクトを開いたとき、メインメニューでCtrl+Lショートカットを押すと、現在表示しているなら閉じ、表示されていないなら表示します。

翻訳対象ファイル一覧ウィンドウの機能:

  • プロジェクトにある翻訳可能なファイル全てのクリック可能な一覧です。これはソースファイルフォルダにある、OmegaTが扱えるファイルです。ファイルをクリックすることで翻訳のために編集ペインに表示します。
  • 各ファイルの分節の数はファイル名の隣に表示されます。
  • プロジェクト全体で繰り返しを除いた(異なった)分節の数。
  • すでに翻訳済みの繰り返しを除いた分節の数。これは分節を翻訳する毎に更新されます。

繰り返しを除いた分節の集合は、全ての分節を取り出して、大文字小文字まで同じ重複した分節を取り除いたものです("Run"と"run"は別のものと考えます)。

「分節の数」と「繰り返しを除いた分節の数」の差を見れば、文章中にどれくらいの繰り返しがあるかが分かります。この数値は、どれくらい繰り返しが重要かについてはっきりと示すことはありません:比較的長い分節が何回も繰り返しているかもしれません(すごくラッキー!)し、単にキーワードが表いっぱいに並んでいる(あまりラッキーでない…)だけかもしれません。

分節化規則を修正することで分節/繰り返しを除いた分節の数を修正することができます。このような修正はプロジェクトの翻訳を始めた後では「決して」行うべきではありません。更なる情報については分節化規則を参照して下さい。

「原文ファイル追加」ボタンを押すことでプロジェクトに原文ファイルを追加できます。選択したファイルを/sourceディレクトリにコピーし、新しいファイルを読み込むためにプロジェクトを再読み込みします。

検索ウィンドウ

文字列検索ウィンドウを使ってプロジェクトの特定の分節を探すことができます。同時に複数の検索ウィンドウを開くことができます。新しいウィンドウを開くには、メインウィンドウでCtrl+Fを使います。

このウィンドウには以下のものがあります:

  • 検索文字列やキーワードを入力するための検索フィールド
  • 検索設定を変えるためのチェックボックスやラジオボタン
  • クリック可能な結果表示エリア

検索フィールドに文字列を入力した後検索ボタンを押すと、プロジェクト内で検索文字列を含む分節を全て表示します。OmegaT は等価な分節を全て一つの実体として扱います -- もしそれらのうちの一つを変更すると、同じ分節は全て更新され、最初のユニークな分節が表示されます。

分節はプロジェクトに現れる順番に表示されます。翻訳した分節の場合は上部に原文が、下部に訳文が表示され、翻訳されていない分節の場合は原文のみが表示されます。

プロジェクトファイルの中から文字列が見つかった場合、それをクリックすることで編集用にエディタペインに表示されます。それから検索ウィンドウに戻って、検索された次の文節を修正するために移動することもできます。このようなパターンは大量に用語を更新するときに便利です。

2 種類の検索が可能です:

  • 完全一致検索では、現在のプロジェクト内、またはフォルダやフォルダ階層を指定して、検索対象文字列を検索することができます。また、完全一致検索は、ワイルドカード（「*」と「？」）を対応しています。'*' は文字0個以上にマッチします（例えば、'run*'で検索をかけると、'run'や'run'や'running'にマッチします）。半角の「?」は文字1個にマッチングします（例えば「run?」は、「runs」や「rung」にはマッチングしますが、「run」や「running」にはマッチングしません）。この検索は、原文、訳文の両方に対して行うことができます。
  • キーワード検索では、開いているプロジェクトを調べて、検索ワードを含む全ての分節を任意の順番で返します。

どのディレクトリの中も検索できますが、対応するファイルのみを検索します。

タグ検証

タグ検証ウィンドウ

タグ検証ウィンドウは訳文文書のフォーマットの間違いを検出するために使います。このウィンドウを開くには、Ctrl+Tを使います。

ウィンドウには以下の 3 つの列があります:

  • タグの間違いがあるかもしれない分節へのクリック可能なリンク
  • 元の分節の内容
  • 翻訳された分節の内容

リンクをクリックするとその分節がエディタペインで有効になります。

タグのエラー

ありえるタグのエラーは以下のものです:

  • タグが原文の分節にあったが、翻訳された分節にない。
  • タグは原文の分節にはなかったが、間違って翻訳された分節に挿入された。
  • タグの順序は翻訳された分節では修正されています。

タグ管理のヒント

タグは一般的に元の文書のある種のフォーマッティングを表現しています。元の文書フォーマッティングの単純化はタグの数の削減に大きく貢献しています。使用フォント、フォントサイズ、色などを可能ならば統一することで、翻訳を単純化し、起こりうるタグエラーの数を減らします。

タグに対して出来ることについては、タグ操作の項目を参照してください。

タグが悩みの種で、フォーマットが今の作業にとても重要というわけではないのなら、ほとんどのフォーマットを原文から取り除くことで翻訳が非常に簡単になることを憶えておいて下さい。

OmegaT でタグは表示されてほしいけれどもほとんどのフォーマットが訳文文書には不要の場合、自由に訳文からタグを取り除いてください。この場合、組タグには特別な注意が必要です。組タグの片方だけが残ってしまうと、文書のフォーマットが壊れてしまい、翻訳したOpenOffice.org文書が不正になったり、翻訳したHTMLファイルが読めなくなったりします。

タグはテキスト自身に含まれているので、より少ないタグが含まれる分節を作るために分節化規則を使うことも可能です。これは上級者向け機能で、初心者は使うべきではありません。

ヘルプブラウザ

(このガイドを表示している)ヘルプブラウザはF1を押すか、メインメニューのヘルプ -> 取り扱い説明ガイド...で呼び出せます。

このウィンドウにはマニュアルおよび戻ると目次の 2 つのボタンがあります。

取り扱い説明ガイドは他の章へのリンクのあるHTML文書です。Webブラウザのように、リンクをクリックすることで選択したページが表示されます。

取扱説明ガイドはOmegaTインストールフォルダのdocサブフォルダにあるので、例えば英語文書ならdoc/en/index.hmtlファイルをWebブラウザで開くことで読むことができます。この方法では、内蔵ヘルプブラウザでは受け付けない外部へのリンクを辿ることもできます。

ダイアログ

OmegaT やプロジェクトの設定を変えるためのダイアログが利用可能です。全てのダイアログは別の項目で記述しているので、以下では単に何のために使うかとどうやって呼び出すかだけをまとめます。

  • プロジェクト設定はプロジェクトフォルダと言語を変更するために使います。Ctrl+Eを押すか、メインメニューからプロジェクト ->設定を呼び出すことで開きます。
  • フォント設定は OmegaT がソース、翻訳、参考訳文、用語集を表示するために使うフォントを変更するダイアログです。これはメインメニューの設定 -> フォント設定から呼び出されます。
  • 利用可能なフィルタのそれぞれで扱われるファイルを修正するために、ファイルフィルタオプションを使えます。
  • 分節化設定ダイアログで、文章を文に分節化する方法を変更できます。文単位の分節化はプロジェクト設定ダイアログで有効化/無効化できます。
  • 翻訳入力行設定ダイアログは、分節を移動するときのOmegaTの振る舞いを変更します。
