Установка и настройка

Установка

Установка включает три стадии. Во-первых, установить J2RE (если ещё не установлена). В-вторых, установить OmegaT. В-третьих, настроить. J2RE OmegaT можно устанавливать в любом порядке. Настройка не особенно важна, но облегчает запуск приложения.

О Java Runtime Environment (среде выполнения Java)

Java Runtime Environment (J2RE) позволяет запускать в системе программы, написанные на Java. Её можно загрузить как отдельный продукт и в составе инструментария разработчика Java.

Java также доступна в виде подключаемого модуля к браузерам. Эта форма Java не позволит запустить OmegaT.

JavaScript — совсем иной продукт, технически совершенно не связанный с Java.

Если вы хотите использовать OmegaT, обязательно нужно установить J2RE в вашей системе. Без неё OmegaT не будет работать. Не следует запускать OmegaT без J2RE. Это напрасная трата времени!

Если вы не знаете, установлена ли J2RE в вашей системе, просто попробуйте установить и запустить OmegaT (указания по установке OmegaT приведены ниже). Если ничего не происходит или появляются сообщения об ошибках, среда выполнения Java, вероятнее всего, не установлена. В этом случае следует её получить и установить.

(Примечание: чтобы запутать клиентов, Sun постоянно меняет названия продуктов. Так, все версии JRE выше 1 известны как «Java 2», поэтому Java Runtime Environment теперь называется J2RE. Java Runtime Environment может также называться Java Virtual Machine или JVM, а на Macintosh она может быть известна как MRJ. Инструментарий разработчика известен как Java Developer's Kit (JDK), Software Developer's Kit (SDK), а теперь также J2SE, J2EE и J2ME, в зависимости от версии. По крайней мере, я так думаю. Но я сам запутался.)

Установка Java Runtime Environment (среды выполнения Java)

Единственная платформа, не требующая Java Runtime Environment, — это MacOSX. MacOSX поставляется с предварительно установленной и сконфигурированной JRE.

Пользователям Windows: узнать, какая версия Java Runtime Environment установлена, можно, щёлкнув по кнопке Пуск / нажав клавишу Windows и выбрав Настройка > Панель управления. Если J2RE уже установлена, будет видна панель управления с подписью «Java(TM)2 Runtime Environment, Standard Edition 1.3.0_01» или сходной.

Пользователям Linux: просто введите Java -version в командной строке. Будет возвращена установленная версия (конечно, если она установлена на системном уровне).

J2RE можно загрузить по ссылке ниже. Она бесплатна (обратите внимания на условия лицензии), но правила именования могут несколько затруднить поиск нужной версии. Понадобится версия для вышей платформы (Linux, Windows, и т.п.). Размер J2RE — около 14 Mб.

OmegaT 1.4.4 должна работать с любой версией J2RE от 1.2 и позднее. Однако мы рекомендуем получить последнюю версию J2RE, поскольку предыдущие версии Java могут не поддерживать определённые функции.

В качестве альтернативы J2REможно установить инструментарий разработчика. Инструментарий разработчика содержит собственную копию J2RE, а также позволяет изменять OmegaT.

Пользователи Windows и Linux могут загрузить Java Runtime Environment с сайтов:

Sun (все платформы)

Blackdown (Linux)

Предыдущие версии OmegaT работают в MRJ для Macintosh OS 9. Версия 1.4.4 может работать, но она не тестировалась на этой платформе. Загрузить MRJ можно с сайта Apple.

Установите J2RE согласно прилагаемым указаниям.

Установка OmegaT

Все платформы

Просто распакуйте содержимое архива в директорию по выбору. Архив может быть в формате *.zip или *.tar.gzip. Пользователи Linux и Mac (Stuffit) не должны испытывать проблем в любом случае; пользователи же Windows могут не иметь утилиты для последнего.

Только Windows

Пользователи Windows могут загрузить установщик для Windows в виде исполняемого файла OmegaT_1.4.4_setup.exe, который просто нужно запустить. В таком случае установщик создаст ярлык для запуска OmegaT в меню Пуск > Программы.

Настройка (только Linux)

Если вы используете Linux, но не очень хорошо его знаете, вот совет, как можно сделать более лёгким запуск OmegaT:

Создайте shell-скрипт, написав текстовый файл следующего содержания:

#!/bin/bash
cd
/java -jar OmegaT.jar

где — директория, содержащая файл OmegaT.jar, а — директория, содержащая исполняемый файл J2RE (который называется java). Если установлена переменная пути Java, эту директорию не нужно указывать, достаточно java -jar OmegaT.jar. Указывать директорию нужно, если не определена переменная пути Java, а также если в системе установлено несколько версий Java, и нужно обеспечить, чтобы OmegaT запускалась с нужной версией.

Сохраните текстовый файл как OmegaT и сделайте его исполняемым (напр., в Konqueror контекстное меню > Properties > Permissions > Exec). Теперь для запуска OmegaT можно просто ввести OmegaT в командной строке. При желании можно поместить значок на рабочий стол, назначив ему исполнение файла скрипта.

Запуск приложения

Macintosh OS X

Щёлкните по файлу OmegaT.jar в программе управления файлами.

Windows

Если OmegaT установлена с помощью установщика для Windows (см. выше), её можно запустить, щёлкнув по ярлыку «OmegaT 1.4.4» в меню Пуск > Программы > OmegaT. Можно также щёлкнуть по файлу OmegaT.exe, OmegaT.bat или OmegaT.jar в файловом менеджере.

Linux

Введите java -jar OmegaT.jar в командной строке. Соблюдайте регистр буке в OmegaT.

Если Java не настроена на определение местонахождения файлов OmegaT, сначала следует перейти в директорию, содержащую файл OmegaT.jar.

Если не установлена переменная пути Java, следует указать полный путь. Если не сделано ничего из перечисленного выше, запуск должен происходить примерно следующим образом:

cd /home/user/OmegaT
/home/user/j2re1.4.2/bin/java -jar OmegaT
