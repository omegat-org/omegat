Linux (Intel) 用户

下载哪个安装包？

安装了与 SUN's Java 1.4 JRE 兼容的 Java 实现了吗？

是的: 

下载 OmegaT_2.0.X_Without_JRE.zip。该安装包可用在任何安装了 Java 1.5 JRE 兼容 JRE 的平台上。

否 / 我不知道:

下载 OmegaT_2.0.X_Linux.tar.gz. 该安装包捆绑了 SUN 的 Java 运行环境。该 JRE 不会与系统中可能已安装的其它 Java 实现相冲突。

OmegaT 安装

要安装 OmegaT，只需解包或解压缩所下载的文件。该操作将会在工作目录中创建一个 omegat/ 目录，在该目录中可以找到运行 OmegaT 所需的所有文件。要解压 .tar.gz 文件：

$ tar xf downloaded_file.tar.gz

向菜单（KDE）或面板（Gnome）中增加 OmegaT

KDE 用户

可按下列步骤向菜单添加 OmegaT ：

  • ［控制中心］－［桌面］－［面板］－［菜单］－［编辑 K 菜单］－［文件］－［新菜单项／新子菜单］。

  • 然后选择适当的菜单，通过［文件］－［新子菜单］ 和［文件］－［新项目］ 添加一个子菜单／项目。输入 OmegaT 作为新菜单项的名称。

  • 在 "命令［Command］" 字段中，通过导航［navigation］按钮找到 OmegaT 启动脚本(在解压目录中命名为 OmegaT 的文件)并加以选中。

  • 点击图标按钮（在 Name/Description/Comment 字段的右边），其它按钮［Other Icons］ - 浏览［Browse］，导航至 OmegaT 应用程序文件夹的 /images 子文件夹。选择 OmegaT 图标。

  • 最后，通过“文件-保存”［File－Save］将所作变更保存。

Gnome 用户

可通过下列步骤将 OmegaT 添加到面板（屏幕顶部的菜单条）：

  • 在面板单击鼠标右键 － “添加到面板”［Add New Launcher］。

  • 在［名称］（Name）字段输入 "OmegaT" ，在[命令] （Command）字段使用［导航］按钮找到 OmegaT 启动脚本(解压文件夹中名为 OmegaT 的文件)。选中并通过［确定］（OK）按钮进行确认。

  • 点击图标按钮，然后点击【浏览】（Browse）……然后导航至 OmegaT 应用程序文件夹的 /images 子文件夹。选中 OmegaT_icon.png 。GNOME 可能无法显示当前格式的图标文件，原本看起来预期值为一份 SVG 文件，但如果选中了该文件夹，将会显示该文件并可以选择 OmegaT_icon.png 。

运行 OmegaT

OmegaT 安装好后，可直接命令行启动，或创建一份包含命令行启动参数的脚本，或直接点击 OmegaT.jar 启动它。根据发行版本方法有所不同。请确认正确设置了 PATH 以及 .jar 与 Java 启动器正确关联。

Windows 用户

下载哪个安装包？

安装了与 SUN's Java 1.4 JRE 兼容的 Java 实现了吗？

是的:

下载 OmegaT_2.0.X_Windows_without_JRE.exe. 该安装包可用在任何安装了 Java 1.4 JRE 兼容 JRE 的平台上。

否 / 我不知道:

下载 OmegaT_2.0.X_Windows.exe. 该安装包捆绑了 SUN 的 Java 运行环境。该 JRE 不会与系统中可能已安装的其它 Java 实现相冲突。

OmegaT 安装

要安装 OmegaT，仅需双击所下载的程序，并按照指示操作。

向启动菜单或桌面添加 OmegaT

在安装过程中，安装程序将询问是否要在 开始 菜单中创建一个文件夹，以及是否想在桌面和快速启动栏创建快捷方式。

在完成安装之后你仍然可以创建快捷方式，只需将 OmegaT.exe 拖动到桌面或 开始 菜单，已将其连接到该处。

运行 OmegaT

一旦安装好 OmegaT ，您可以通过点击 OmegaT.jar 或者从命令行启动它。

然而，最简单的方法是运行 OmegaT.exe 程序。在此情形下，将从OmegaT.l4J.ini文件读取启动参数，该文件与可执行文件在同一目录，你可对它进行修改以反映你的设置。下面的 INI 文件将保留 1GB 内存，以法语为用户语言，将国家设置为葡萄牙：

# OmegaT.exe runtime configuration
#
-# 要使用某参数，仅需删除 '-' 前面的 '#'
# Memory
-Xmx1024M
# Language
-Duser.language=FR
# Country
-Duser.country=PT
# Anti-aliasing (Java 1.4 and 1.5 only)
#-Dswing.aatext=true
# Entity expansion for XML (Java 1.4 only)
#-DentityExpansionLimit=400000

MacOSX 用户

下载哪个安装包？

OmegaT 2.0 的运行需要 Java 1.5。Mac OSX版10.4（Tiger）的用户应该确保在安装 OmegaT 前使用了 Java 的最新版本。从苹果菜单调用 "软件更新（Software Update）..." 以更新 Java 的版本。

下载 OmegaT_2.0.x_Mac.zip 。

OmegaT 安装

双击 OmegaT_2.0.x_Mac.zip 解压缩。该操作将创建一个名为 OmegaT ，且内有 OmegaT 图标的文件夹。该文件夹包含两份文件： documentation.html 和 OmegaT.app 。将文件夹复制到合适的文件夹（如：Applications)。一旦完成上述操作，即可删除文件 OmegaT_2.0.x_Mac.zip 。

将 OmegaT 添加到 Dock

将 OmegaT.app 拖放到 Dock 即可将 OmegaT 添加到 Dock。

运行 OmegaT

双击 OmegaT.app 或点击它在 Dock 中的位置。

如果向修改 Java 的运行参数，可以通过编辑 OmegaT.app/Contents/ 内的文件 Info.plist 修改 OmegaT.app 的默认设置 。右键点击并选择文本编辑器，打开文件 "Info.plist" 。您可以修改 Properties 部分以及 VMOptions 部分。如果要向 VMOptions 添加值 ，在两者之间放置一个空格。例如， - Xmx1024M - Duser.language = ja 将提供 1024MB 内存，并以日文用户界面启动 OmegaT。

直接启动 OmegaT.app 将不允许运行多个 OmegaT.app 实例。如果您需要运行多个 OmegaT.app 实例 ，使用位于 OmegaT.app /Contents/MacOS/ 的文件 JavaApplicationStub 以代替前者。

无法从 OmegaT.app 安装包中使用 Lucene stemmer。您需要从命令行运行 OmegaT 。（详见下文）

要改变命令行中的 Java 启动参数，可调用位于 OmegaT.app/Contents/Resources/ Java/ 的 OmegaT.jar 文件。从命令行启动 OmegaT.app 的效果等同于使用 OmegaT 的“通用”运行包。您将可以运行多个 OmegaT 的实例，如同使用 Lucene stemmer 。查阅下文中的 "命令行启动 " 了解更多信息。

要访问 OmegaT.app 运行包中的某个位置，可右键点击 OmegaT.app 并选择 "【显示运行包内容】Show Package Contents" 或者从命令行直接 "cd" 到那里。

Mac OSX 的优势

可以从 Mac OSX 提供的服务直接访问 OmegaT.app。因此，你可以在 OmegaT 的任何地方选择一个单词，然后使用服务在 Spotlight 或谷歌提供的字典查阅该词。你还可以使用 AppleScript 或 Automator 创建能够自动频繁查询的服务或脚本。

安装便捷的快捷方式

按照系统的提示在方便的地方安装 OmegaT 快捷方式。

运行 OmegaT

OmegaT 安装好后，可直接命令行启动，或创建一份包含命令行启动参数的脚本，或直接点击 OmegaT.jar 启动它。根据发行版本方法有所不同。请确认正确设置了 PATH 以及 .jar 与 Java 启动器正确关联。

命令行启动

正常情况下，没有必要从命令行启动 OmegaT 。然而，从命令行启动 OmegaT 的方法使得程序的行为方式变得可控，也就是说可以修改。有两种使用命令行启动 OmegaT 的方法。

方法1：打开命令行窗口

命令行窗口也被叫做“终端窗口“。在 Windows 中可以称之为 "MS-DOS 窗口"，可从开始菜单，程序中的 "MS-DOS" 项找到。Mac OSX 对应的是位于 应用程序［Applications] → 实用工具［Utilities］ 中的应用程序终端。

为启动 OmegaT，通常你必须执行两条命令。第一条是：

cd {文件夹}

其中 {文件夹} 是 OmegaT 程序，特别是OmegaT.jar文件所处的文件夹的名称（带有完整路径）。实际上，该命令可能是这样的：

在 Windows 上
cd C:\Program Files\OmegaT
在 Mac OSX 上
cd <OmegaT.app location>/OmegaT.app/Contents/Resources/Java/
在 Linux 上
cd /usr/local/omegat

该命令将目录改为包含可执行 OmegaT 文件的目录。第二条是实际启动 OmegaT 的命令：就最基本的形式而言，该命令为：

java -jar OmegaT.jar

请注意大小写－－这很重要。

该方法对于调试有特别的意义：如果在使用程序过程中发生了错误，在终端窗口所显示的错误信息对查找原因提供了有益的信息。

方法2: 修改 .bat 文件/启动脚本。

方法1对于经常性启动一个程序来说是不切实际的。为此，上述两条命令被包含在一份文件（一份脚本、在 Windows 上也叫做一份 ".bat 文件）。

执行该文件时，所包含的命令将自动执行。因此，要修改启动命令，修改该文件即足以。如果是从文件启动 OmegaT ，将执行新的命令。

启动命令参数

基本的参数已经在上面提到过了。修改命令将涉及到增加额外的“参数”。这些内容在 命令行参数 有详细描述。

从源代码编译 OmegaT

下载哪个安装包？

下载 OmegaT_2.0.x_Source.zip
该安装包包含了对应发布版本的源文件，可以在任何安装了 Java 1.5 JDK 或其兼容 Java 系统的及其上使用。您还需要 ant 工具来编译 OmegaT。
OmegaT 使用 SubVersion 进行版本控制。最新的代码可在 SVN 服务器找到：
https://omegat.svn.sourceforge.net/svnroot/omegat/trunk

编译 OmegaT

解压 OmegaT_2.0.X_Source.zip 文件，并进入 OmegaT_2.0.X_Source 目录或者进入 SVN 检出的 ./omegat/ 目录。检查 build.xml 文件是否在该目录中。然后，在命令行输入：

$ ant jar release

该命令将在 ./dist/ 目录中创建一份完整的 OmegaT 发行文件，其中包含运行 OmegaT 需要的所有文件。

使用 Java Web Start

使用 Java Web Start技术（对Java 1.5以上版本），只需一次点击就可通过网络部署独立的 Java 应用软件。Java Web Start能够保证应用程序的最新版本以及正确 Java运行环境（JRE） 版本得到部署 。为首次通过 Java Web Start 启用 OmegaT，在浏览器中输入下面的 URL：
    http://omegat.sourceforge.net/webstart/OmegaT.jnlp

下载文件 OmegaT.jnlp 然后点击它。在安装过程中，根据操作系统的不同，可能会收到几条安全警告。该证书由 "Didier Briel" 自行签署。您赋予该版本的许可（可能会被认为是“对计算机的无限制访问”）和您给予本地版本的一样的，也就是说它们被允许访问计算机的硬盘驱动器。如果处于在线状态，之后再点击 OmegaT.jnlp 将会检查是否有升级，如果有的话将安装升级包并运行 OmegaT。初始化安装之后，你当然可以在离线状态使用 OmegaT.jnlp 了。

隐私政策： OmegaT 的 Java Web Start 不会在您的计算机之外保存您在之上运行的任何资料。该应用程序只在您的计算机上运行。您的文档和译法记忆将保留在您的计算机上，并且 OmegaT 项目就没有你的工作或信息。

请注意，如果您需要某个启动命令参数（见上文），就不得不使用正常安装。
