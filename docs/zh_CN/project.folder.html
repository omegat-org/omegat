OmegaT 项目是一个文件夹，其中包含若干文件和子文件夹。

source

source 子文件夹包含要翻译的文件。之后您可以将文件添加到其中。注意：source 子文件夹的结构可以随您喜好采用任何形式。如果要翻译的文件是树形结构的一部分（比如像网站中），你只需要指定顶级子文件夹，OmegaT 将维护整个内容同时维持树形结构不变。

target

该子文件夹一开始是空的。要向其中添加内容，请选择项目 → 创建译文文档 ( Ctrl + D )。位于 source 文件夹下的文件，无论是否已翻译，都会在此处生成，维持着与 source 子文件夹中相同的层次结构。target 子文件夹的内容将反映当前的翻译状况，正如保存在 当前 /omegat/project_save.tmx 中的项目翻译记忆库所体现的那样。因此，未译片段会维持原文语言。

tm

/tm/ 文件夹可以包含任意数量的辅助翻译记忆库——即 tmx 文件。此类文件可以使用前文所述三种变体格式任意其一。注意其他 CAT 工具可以输出（也能输入）tmx 文件，通常还能支持全部三种格式。当然，最好是使用 OmegaT 专用的 TMX 文件（见前文），这样能够保留片段中的内联格式。

tm 子文件夹中的翻译记忆库的内容用于为待翻译的文本生成建议。任何已翻译并存储在这些文件中的文本只要与当前正在翻译的文本足够相似就会出现在模糊匹配中。

如果辅助翻译记忆库中有与待译文本完全相同的原文片段，OmegaT 会按照选项 → 编辑行为...对话框窗口中定义的行为进行操作。例如（如果按默认值），使用来自辅助翻译记忆库的翻译并冠以 [模糊] 前缀，这样译者可以在后面检查带有此标签的片段是否已正确翻译。

有可能会出现这种情况：在 tm 子文件夹中的翻译记忆库中包含多个原文文本完全相同但译文不同的片段。TMX 文件是按照名称顺序进行读取的，并且对于给定 TMX 文件是逐行读取其中的片段的。因此，对于有着完全相同的原文文本的片段，会采用最后一个（注意：显然，先考虑避免出现这种情况会更有意义)。

注意，在 tm 文件夹中的 TMX 文件可以用 gzip 进行压缩。

tm/auto

如果从一开始就非常确定某些翻译记忆库中的翻译是完全正确的，则可以将其放入 tm/auto 文件夹，这样就无需对大量 [模糊] 进行确认。

  1. 将 TMX 文件放入 /tm/auto 中。

  2. 打开项目。更改会体现出来。

  3. 在项目中任意位置稍做改动。这会修改 project_save.tmx（添加来自“自动”TMX 的合适的翻译单元）

注意：如果在第 3 步之前就从 /tm/auto 中删除了 TMX，则不会添加任何其他翻译单元。

tm/enforce

如果你确信某个 TMX 比 OmegaT 的 project_save.tmx 更靠谱，请将此 TMX 放到 /tm/enforce 中来无条件地覆盖现有的默认翻译。

  1. 将 TMX 放入 /tm/enforce 中。

  2. 打开项目。更改会体现出来。

  3. 在项目中任意位置稍做改动。这会修改 project_save.tmx。

  4. 确定被强制更改的片段的豁免权：

    • 如果它们 不需要 豁免后继更改，请将 TMX 从 /tm/enforce 中删除。

    • 如果它们 需要 豁免后继更改，请将 TMX 保留在 /tm/enforce 中。

注意：如果在第 3 步之前就从 /tm/enforce 中删除了 TMX，则根本不会保存任何强制修改。

tm/mt

在编辑器窗格中，如果插入的匹配项来自名为 mt 的文件夹中的 TMX 文件时，当前所在片段的背景会变为红色。离开此片段后背景恢复正常。

tm/penalty-xxx

有时，将由于题材、客户、修订状态等原因而不太可靠的翻译记忆库与高质量的翻译记忆库区分开来会很有用。对于放在名为 penalty-xxx（xxx 在 0 到 100之间）的文件夹中的翻译记忆库，匹配度将根据文件夹的名称而降低：位于名为（例如）penalty-30的文件夹中的任意翻译记忆库内匹配度为 100% 的匹配项其匹配度会降到 70%。三种匹配度百分比都会施加罚分：匹配度 75、80、90 在这个案例中会降为 45、50、60。

dictionary

一开始是空的，此文件夹将包含您添加入项目的词典。关于此主题的更多信息，请参阅词典。

glossary

此文件夹一开始是空的。它将包含你在项目中使用的词汇表。关于此主题的更多信息，请参阅词汇表。

omegat

omegat 子文件夹内至少会包含一个文件，可能还有若干其他文件。这里面最重要的文件是 project_save.tmx，他是项目的有效翻译记忆库。随着翻译进程，在每次翻译会话开始和结束时，此子文件夹中会逐步添加此文件的备份（扩展名为 bak）。

在翻译中，该子文件夹中还可能会创建其他文件，如下所示：

stats.txt

包含当前项目的当前统计数据。您可以选择工具 > 统计数据来查看它。

ignored_words.txt., learned_words.txt

由拼写检查器创建和使用。如果您已经有了希望拼写检查其忽略/接受的单词集，只需将相应的两个文件复制到当前项目的 omegat 子文件夹中。

project_stats_match.txt

包含最新的项目匹配统计数据，由工具>匹配统计数据创建

segmentation.conf

如果存在，则包含项目专用的片段分割规则。

filters.xml

如果存在，则包含项目专用的文件过滤器。

uiLayout.xml

如果存在，则包含项目专用的图形界面设置。

omegat.project（文件）

包含项目参数在项目属性对话框中定义的项目参数。

.repositories

在团队项目中，此文件夹包含直接连接到远程服务器的项目树结构的版本副本。

你可以用此文件夹和 Git 或 SVN 客户端对远程文件进行更改（例如，删除某个文件）。

在某些操作系统中，默认情况下不显示此文件夹。激活“显示隐藏文件”选项来使其可见。
